# A092 v2 外部 Validator 最小合同

外部 Validator 必须在候选执行前冻结，并位于候选解生成过程之外。候选评价器与外部适配器可以实现同一题意公式，但不得调用同一函数、共享同一实现文件或具有相同实现哈希。

每次验证必须记录：输入文件、候选解、候选评价器、外部适配器和本合同的 SHA256。数据契约审计必须覆盖合并单元格、缺失值、单位换算、聚合键、时间槽顺序、历史边界状态和 2–3 个手算 fixture。

判定顺序固定为：

1. 外部适配器未预先冻结、实现不独立或数据契约审计失败，判 `experiment_invalid`。
2. 外部验证基础设施有效，但目标复算失败或任一硬约束失败，判候选解无效。
3. 只有目标与全部硬约束同时通过，才允许论文使用目标值和改进率；最优性强结论还必须另有严格证明、合法求解器证书或完整有限空间穷举证据。
