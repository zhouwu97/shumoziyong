# 2024-C 冻结数学基准合同

本文件只公开 2024-C 冻结数学复核在执行前必须知道的输入、输出和评价口径；不公开 Validator 的实现细节。

## 1. 输出合同

`formal_result.json` 的顶层字段为 `scenarios`，其值是列表。每项必须包含：

- `scenario_id`：`q1_waste`、`q1_discount`、`q2_frozen` 或 `q3_frozen`；
- `assignments`：每项含 `plot_id`、`year`、`season`、`crop_id`、`area_mu`；
- `objective_reported`：按本合同重算出的目标值（元）。

## 2. 销售上限与目标口径

- 销售上限键为 `crop_id + season`。
- 同一作物、同一季次的生产量必须跨全部地块和地块类型汇总后，再与对应预期销售量比较。
- Q1 `q1_waste` 的超额产量收入为 0；`q1_discount` 的超额产量收入为相应单价的 50%。

## 3. 轮作口径

- 智慧大棚按连续季次检查：第一季 → 第二季 → 下一年第一季。
- 其他地块按同一可用季次的相邻年份检查；水浇地、普通大棚等仍须遵守其季次适配和模式规则。
- 2023 历史状态进入 2024 首次检查；智慧大棚还须比较 2023 第二季与 2024 第一季。

## 4. Q2/Q3 冻结代表场景

令 `k=year-2023`。`q2_frozen` 与 `q3_frozen` 使用相同的确定性代表参数：

- 小麦、玉米预期销售量：乘以 `1.075^k`；其他作物销售量保持 2023 基线；
- 全部亩产量：乘以 `0.95`；
- 全部种植成本：乘以 `1.05^k`；
- 粮食价格保持 2023 基线；蔬菜价格：乘以 `1.05^k`；
- 羊肚菌价格：乘以 `0.95^k`；其他食用菌价格：乘以 `0.97^k`；
- 超额产量按滞销处理。

Q3 的相关性和风险改善不进入冻结目标，必须另行提供可复现随机样本和比较证据。

## 5. 边界

本合同用于冻结数学基准复核。它不代表 Q2/Q3 的唯一学术建模方法；随机优化、相关性模拟和风险分析可作为补充实验输出，但不得替代 `q2_frozen` 或 `q3_frozen` 的 `objective_reported`。
