# 新题比赛运行契约

本文件已经与基础规则、当前 Profile、已审核 Plugin、允许进入正式包的 Patch 及 Gate 0-5 检查表编译为同一个 Runtime Pack。执行代理只能依据该单一运行包和已冻结的题面材料工作，不得再查找仓库中的其他相对路径或历史文件。

## 不可突破的边界

1. 第一轮只执行 Gate 0：读题、材料诊断、题型判断、候选路线、风险与人工确认项；不得写完整论文、正式代码或最终答案。
2. 未获得人工确认不得进入下一 Gate。`emergency` 只减少暂停点，绝不减少 Gate 业务产物、身份绑定、哈希校验或最终验收。
3. `review_ready` Patch 不得用于正式比赛。只有运行包中已经编译的正式 Patch 可以使用。
4. `new_problem` 的证据用途是 `competition_execution`，不能自动构成 Patch 首级晋级证据。
5. 每个 Gate 产物必须绑定当前 Run ID、Problem ID、Profile、Runtime Version 与 Runtime Pack SHA-256；材料、运行包、状态或 Seal 不一致时立即停止。

## Gate 顺序

- Gate 0：诊断题面和冻结材料，输出人工确认项。
- Gate 1：确认模型路线、变量、约束、基线与验证方式。
- Gate 2：确认实现模块、输入输出、测试和降级策略。
- Gate 3：确认结果、约束、比较与稳健性。
- Gate 4：只将已有结果和证据映射为论文论点。
- Gate 5：从评委视角完成可复现性、风险和交付验收。

任何 Gate 失败、材料漂移或人工审核未通过时，不得跳过、倒退篡改记录或继续推进。
