# 五、当前论文最需要解决的数学问题

这比图表和语言更优先。

## 问题二没有在全部 402 家供应商中寻找成本最低的 26 家组合

当前程序先按照净产品能力排序，取能力最大的前 26 家，然后只在这 26 家内部优化采购成本和运输损耗。代码中的候选集合由排序累加直接确定。

后续 MILP 也是基于这一固定候选集合建立。

这能够证明：

* 任意 25 家能力不足；
* 存在 26 家可行；
* 最少供应商数为 26。

但不能证明：

> 当前方案是在全部可行的 26 家组合中采购成本最低、损耗最小的方案。

### 应改成完整三阶段 MILP

保留全部 402 家供应商及二元变量 (z_i)。

第一阶段：

[
\min \sum_i z_i
]

得到：

[
K^*=26
]

第二阶段加入：

[
\sum_i z_i=26
]

然后在全部供应商中：

[
\min \sum_{i,t} c_i s_{it}
]

第三阶段锁定采购成本后：

[
\min \sum_{i,j,t}\ell_jx_{ijt}
]

这样才能真正宣称：

> 在最少 26 家供应商的所有可行组合中，获得词典序最优的采购与运输方案。

这是当前第一优先级，重要性高于增加图表。

# 二、P0：问题二并不是真正的全局词典序优化

这是目前最关键的问题。

## 论文实际采用的流程

第 13—15 页及附录第 29 页的程序逻辑是：

1. 按放松后的净产品能力排序；
2. 直接取能力最大的前 26 家作为候选集合；
3. 固定这 26 家；
4. 只在这 26 家内部最小化采购成本；
5. 再在成本最优条件下最小化损耗。

代码中明确是：

```python
candidates = choose_minimum_supplier_set(adjusted, demand, loss_multiplier)
builder, ... = _base_builder(adjusted, candidates, loss_multiplier)
```

也就是说，成本阶段和损耗阶段已经没有 402 家供应商的选择变量了。

## 为什么这会影响结论

前 26 家能力最大的供应商，确实构成一个可行的 26 家集合。

但可能还存在另一个组合：

* 也是 26 家；
* 总能力稍低但仍能满足需求；
* A、B、C 类结构更便宜；
* 更适合低损耗承运商的容量分配；
* 最终采购成本或运输损耗更低。

当前模型没有搜索这些组合。

因此论文目前只能说：

> 在按净能力排序选出的这一组 26 家供应商中，所得方案的采购成本和运输损耗分别最优。

不能说：

> 在所有仅使用 26 家供应商的可行方案中，该方案采购成本和运输损耗全局最优。

标题、摘要和正文中“词典序 MILP”“在最少数量下最小化采购成本和损耗”的表述因此偏强。

## 正确的模型应当怎样建立

问题二应直接在全部 402 家供应商上保留选择变量：

[
z_i\in{0,1},\qquad y_{ij}\in{0,1},\qquad x_{ij}\ge0.
]

核心约束为：

[
\sum_jx_{ij}\le u_i z_i,
]

[
\sum_i x_{ij}\le6000,
]

[
\sum_jy_{ij}\le z_i,
]

[
x_{ij}\le u_i y_{ij},
]

[
\sum_{i,j}\frac{(1-\ell_j)x_{ij}}{q_{k(i)}}\ge28200.
]

再执行真正的三阶段词典序优化：

### 阶段一

[
\min\sum_i z_i.
]

得到：

[
\sum_i z_i=26.
]

### 阶段二

固定供应商数量：

[
\sum_i z_i=26,
]

再求：

[
\min\sum_{i,j}c_{k(i)}x_{ij}.
]

### 阶段三

固定最优采购成本，再求：

[
\min\sum_{i,j}\ell_jx_{ij}.
]

这样才能证明：

* 26 家是最少数量；
* 在所有 26 家方案中采购成本最低；
* 在最低采购成本方案中损耗最低。

### 对当前结果的影响

预计“26 家”这一数量不会改变，因为上下界证明已经闭合。

但以下内容可能全部变化：

* 具体供应商名单；
* A、B、C 类使用量；
* 周相对采购成本 20,463.567522；
* 周损耗 102.938185；
* 各转运商负载；
* 与贪心基线的比较结果。

因此这不是只修改文字就能解决的问题，需要重新求解问题二。

# 三、压力测试中的“不可行”结论也没有充分证明

这个问题与问题二的固定候选集缺陷直接相关。

第 20 页报告：

* 需求增加 10%：不可行；
* 关键供应商全期中断：不可行；
* 关键能力下降 10%：限时未知。

但从附录程序看，压力测试仍然是：

1. 根据放松能力得到理论最少供应商数量；
2. 固定能力排序最靠前的这组供应商；
3. 如果该组正式 MILP 无解，就返回失败。

它没有继续检查：

* 换一组相同数量的供应商是否可行；
* 增加一两家供应商是否可行；
* 使用全部供应商时网络是否仍不可行。

因此，按照附录实现，当前“不可行”最多只能解释为：

> 按能力排序得到的当前最小候选集合，在正式运输和单承运商约束下不可行。

它不能自动推出：

> 全部 402 家供应商构成的网络在该情景下均不可行。

尤其是需求增加 10% 时，即使最小基数集合不可行，增加供应商数量后仍可能恢复可行性。

1. 先重新最小化供应商数；

4. 如果只达到时间上限，应报告：

   * 是否找到可行解；
   * 当前下界；
   * 当前上界；
   * MIP gap；
   * 而不是只写“未知”。

特别是关键能力下降 10% 只给 5 秒时间，证据过弱。正式论文中至少应使用与主模型相同的时间上限，或者用 LP 放松与可行性恢复模型辅助判断。

# 四、供应能力参数的统计含义存在概念问题

论文定义：

[
u_i=\bar S_i^+p_i.
]

其中：

* (\bar S_i^+) 是正供货条件下平均供货量；
* (p_i) 是已订购周发生正供货的概率。

因此 (u_i) 本质上近似表示：

[
E(S_i\mid O_i>0),
]

也就是“下单周的历史期望供货量”。

它不是严格意义上的“供货上限”。

但第 11、14 页将它称为“常规期望供货上限”，并作为硬约束：

[
s_{it}\le u_i.
]

“期望值”和“上限”在统计意义上并不是同一个概念：

* 实际供货可能高于期望值；
* 也可能远低于期望值；
* 期望值没有给出任何保证概率；
* 它既不是物理产能上限，也不是置信下界。

论文已经多次说明它不是未来保证值，这是好的；但在模型中又把它作为确定性可用能力，仍然会导致结果过于乐观。

## 风险在结果中已经显现

问题二方案中：

* 每周产品等价到货量恰好为 28,200；
* 每周期末库存恰好贴着 56,400 的安全下界；
* T3、T6 满载；
* T2 负载达到 99.28%。

也就是说，在模型期望值下几乎没有冗余。

只要任何供应商低于期望供货，安全库存就会下降。当前“两周安全库存”只是确定性模型中的账面下界，并不是概率意义上的安全水平。

## 建议至少增加一种稳健处理

较容易落地的方法是设置能力折减系数：

[
u_i^{\text{plan}}=\alpha u_i,
\qquad \alpha\in{0.8,0.85,0.9,0.95,1.0}.
]

重新计算：

* 最少供应商数量；
* 采购成本；
* 运输损耗；
* 最大产能。

更严谨的方法是使用：

* 历史供货量的低分位数；
* 均值置信下界；
* Bootstrap 情景；
* 机会约束；
* CVaR 或两阶段随机规划。

在不大幅扩充篇幅的情况下，建议正文至少增加 (\alpha=0.9) 和 (\alpha=0.8) 两个情景，而不是只对单个关键供应商下降 10%。

# 五、订购量与供货量的线性关系过于简单

第 13 页使用：

[
s_{it}=\rho_i o_{it},
\qquad
\rho_i=\min\left(\frac{\sum S_{it}}{\sum O_{it}},1\right).
]

这个关系意味着：

> 不论订购量大小，只要订购量翻倍，期望供货量也按同一比例翻倍，直到碰到 (u_i)。

但历史响应率通常只在历史订购区间内有效。当前模型没有限制：

[
o_{it}
]

不能超过历史常见订购量。

对于响应率较低的供应商，为了达到 (s_{it}=u_i)，模型可能给出很大的订购量：

[
o_{it}=\frac{u_i}{\rho_i}.
]

这可能超出历史订单范围，属于外推。

## 建议增加约束

最简单的方式是：

[
o_{it}\le O_i^{95%},
]

其中 (O_i^{95%}) 为该供应商历史非零订购量的 95% 分位数。

更合理的是拟合分段线性响应函数：

[
s_{it}\le f_i(o_{it}),
]

使供货量随着订购量增加逐渐饱和，而不是全区间保持固定兑现率。

此外，问题一的能力指标较大程度反映了企业过去“给这家供应商下了多少订单”，未必等于供应商真实的外部供货能力。这一点也应在局限性中明确说明。

# 六、问题三数学结果成立，但运营方案不合格

* 268 家订购量小于 100 m³；
* 存在拆分运输；
* 次要承运商承担 5.0586% 的体积。

论文对此披露得很诚实，所以这个结果不是“算错”，而是目标函数设计的必然结果。

## 主要问题

论文以“80 秒内硬单承运商模型未返回可行解”为理由，改用连续 LP。

但在平稳参数下，问题三只需要求解一个代表性单周模型，再复制到 24 周。模型规模大致只有：

* 402 家供应商；
* 8 家转运商；
* 约 3216 个承运商二元变量。

这个规模并没有大到足以直接放弃 MILP。80 秒未得到结果，只能说明当前建模或求解参数需要优化，不能说明硬单承运商路线不适合。

## 更合理的处理方式

建议问题三改成“最优性—可执行性”两层方案。

先求原始理论最优：

1. C 类最少；
2. 总原料最少；
3. 损耗最少。

得到当前 LP 下界。

再设置一个极小的目标容差，例如：

[
Z_{\text{loss}}
\le Z_{\text{loss}}^*(1+0.1%),
]

然后优化运营指标：

4. 最少启用供应商数；
5. 最少拆分供应商数；
6. 最大次要承运量最小；
7. 设置最小订单量。

例如引入：

[
\delta z_i\le s_i\le u_i z_i,
]

其中 (\delta) 可取 20、50 或由历史订单分位数确定。

这样可以形成：

* 理论损耗最优方案；
* 损耗增加极小但可实际执行的推荐方案。

当前直接把 305 家方案作为问题三最终订购方案，竞赛评审容易认为“数学结果漂亮，但没有真正解决企业执行问题”。

# 七、问题一评价可解释，但稳健性还不够

问题一的四指标设置总体合理：

* 产品等价能力；
* 按单供货概率；
* 正供货稳定性；
* 累计兑现率。

但仍有三个问题。

## 1. 极差标准化受长尾影响明显

第 10 页图 2 中，产品等价能力归一化后的中位数仅为 0.001。

这说明能力指标极度右偏，少数头部供应商拉高了最大值，使大部分供应商的能力得分都挤在零附近。

此时 50% 的能力权重，实际效果可能是：

* 极强地区分最头部少数供应商；
* 对中后部供应商几乎没有分辨能力。

可考虑：

[
C_i'=\log(1+C_i)
]

后再标准化，或者采用分位数标准化。

## 2. 权重敏感性只做了局部扰动

当前只做：

* 单项权重 ±10%；
* 等权方案。

这能说明局部稳定性，但不能说明整个合理权重空间中的稳健性。

建议从权重单纯形随机抽取 1000—5000 组权重，统计：

* 每家供应商进入前 50 的概率；
* 平均排名；
* 排名标准差；
* 前 10 或前 50 的稳定核心集合。

这会比单纯的 Spearman 系数更有说服力。

## 3. 问题一与后续优化衔接较弱

论文反复说明前 50 名只形成候选信息，但问题二实际又在全部 402 家供应商中按能力重新选取，问题一的综合排名没有真正参与后续决策。

这并非数学错误，但会削弱四问的整体叙事。

建议增加一张表：

* 问题一前 50 与问题二最终 26 的交集数量；
* 最终 26 家在综合排名中的分布；
* 未进入前 50 但被问题二选中的供应商及原因；
* 前 50 中未入选供应商的成本、类别或运输原因。

这样能体现评价模型与优化模型各自承担的作用。
