# 本轮闭环执行计划

## 1. 本轮目标

- 目标类型：自动选题 + 第二道 T3/T4 官方旧题泛化测试候选；若无 T3/T4，则先自动获取官方材料，失败后仅做流程冒烟或材料受限测试。
- 是否自动选题：是。用户仅提供了 `B311.pdf` 作为本地候选材料线索，未指定可直接作为官方旧题输入的题面、附件和模板。
- 是否目标为 T3：是，优先寻找非 2023-A、非 2023-B、多题号独立官方工程优化旧题。
- 是否允许计入 stable：本轮只能建议是否具备 stable 候选资格；未经人工确认不正式标记 stable。若本轮为不同题号 T3/T4 且通过，可作为第 2 道 T3/T4 候选。
- 本轮禁止事项：不写完整论文，不写求解代码，不给最终答案，不把优秀论文 PDF 当官方题面，不把 T0/T1/T2 或 M1/M2/M3/M5 风险测试计入 stable，不修改正式 base/plugin/patch。

## 3. 自动选题计划

- 扫描目录：`tests/old_problems/`、`output/closed_loop/`、`papers/`、`tmp/`、`高教社杯全国大学生数学建模竞赛优秀论文/`、项目根目录下 PDF/DOCX/XLSX/CSV/TXT/MD、`official_materials/`。
- 候选题识别方法：按文件名、目录、manifest、旧题记录和扩展名识别官方题面、附件、结果模板、优秀论文、学习卡片、解析/答案风险材料。
- T0-T4 判定方法：T0 为优秀论文/题解/答案来源；T1 为官方题面或附件不完整；T2 为官方材料齐但与 patch 同题；T3 为独立官方题面 + 必要附件 + 输出要求齐全且非同题污染；T4 为至少两道不同 T3/T4 通过后的稳定候选测试。
- M1-M5 判定方法：M1 检查优秀论文/答案/解析泄漏；M2 检查附件或模板缺失；M3 检查与 A092/A127 或已有同题源污染；M4 检查是否照搬特殊公式、数值、布局；M5 检查关键题意是否来自人工补全。
- 候选题排序表输出位置：`output/closed_loop/候选题排序_工程优化_2026-07-05.md`。
- 选题失败时动作：自动尝试获取官方材料；获取失败后生成失败报告和最小人工补充清单。

## 4. 材料资格审查计划

| 检查项 | 判定方式 |
|---|---|
| 官方题面 | 本地 manifest、文件来源 URL、官方/主办/承办/学校镜像页面共同确认 |
| 官方附件 | `attachments/` 中有题面指定的必要数据文件，且来源可追溯 |
| 输出模板 | `templates/` 中有 result/template/submit 等官方输出格式，或题面明确不需要模板 |
| 同题污染 | 与 A092/A127 的 2023-A 定日镜场，以及已有 2023-B T3 同题源隔离 |
| 答案泄漏 | 不把优秀论文、题解、参考答案、博客复述作为 T3 输入 |
| stable 资格 | 仅 T3/T4、无 M1/M2/M3/M5、无 P3/P4/P5，且不同题号累计至少两道后才建议 |

## 5. 官方材料自动获取计划

- 是否需要自动获取官方材料：若本地没有新的 T3/T4，必须执行。
- 搜索目标：官方赛题题面、必要附件、结果模板或提交格式。
- 允许来源：全国大学生数学建模竞赛官网、高教社杯相关官方页、主办方/承办方页面、可信高校镜像、可信国内镜像。
- 禁止来源：优秀论文、获奖论文、参考答案、题解解析、博客复述、论坛讨论、含最终数值结果说明。
- 保存目录：`official_materials/{年份}_{题号}/problem/`、`attachments/`、`templates/`。
- manifest 输出位置：`official_materials/{年份}_{题号}/manifest.md`。
- 获取失败报告位置：`output/closed_loop/官方材料获取失败_工程优化_2026-07-05.md`。
- 获取失败后动作：选择最高质量 T0/T1/T2 做流程冒烟，或停止并列出最小人工材料清单。

## 6. 总控诊断计划

- 输出结构：训练材料理解、材料资格审查、旧题总控诊断、优化类专项检查、模型候选与质疑、数据需求与代码准备、图表与结果计划、人工确认项、100 分评分、失败标签检查、最终判定。
- 工程优化必查项：评价函数/目标函数、优化对象、目标、决策变量、约束条件、评价指标、简单方法优先、智能算法质疑、降维/分区/规则化/参数化、优化前后对比和敏感性分析。
- 禁止输出内容：不写完整论文，不写代码，不给最终数值答案，不读取或复述优秀论文答案。

## 7. 评分与闸门计划

- 评分表：题目理解 15，子问题拆解 15，题型判断 10，模型候选与取舍 20，数据需求 10，建模路线 15，图表与结果计划 10，人工确认项 5。
- P 标签：逐项检查 P1-P10，重点闸门为 P3/P4/P5。
- M 标签：逐项检查 M1-M5，M1/M2/M3/M5 直接阻断 stable 计数。
- 失败复盘触发条件：总分低于 80，或出现 P3/P4/P5，或出现 M4 照搬且影响质量，或错误声明可计入 stable。
- 修复草案触发条件：失败原因来自 base/plugin/patch 约束不足；未经人工确认只写草案。
- 重测任务触发条件：材料等级不是 T3/T4，出现 M1/M2/M3/M5，得分低于 80，或需要第二道稳定候选测试。
- stable 计数条件：至少 2 道不同题号 T3/T4 官方旧题通过，且无 P3/P4/P5/M1/M2/M3/M5。
