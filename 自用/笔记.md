熵权法学习总结
熵权法，它主要是用来确定各个评价指标权重的。主要就是看不同指标的数据差别大不大。如果某个指标在不同对象之间差异比较明显，说明它区分能力比较强，提供的信息比较多，那么这个指标的权重就会更高。相反，如果某个指标的数据都差不多，对结果区分不明显，权重就会比较低。熵权法的好处是比较客观，不是完全靠人为判断哪个指标重要，而是根据数据本身来计算权重。它的一般过程是先整理原始数据，然后对指标进行正向化和标准化，再计算信息熵、差异系数，最后得到各指标的权重。不过它也不是完全没有问题。因为它主要看数据差异，有时候数据差异大不一定代表这个指标在现实中真的更重要。所以在实际使用时，熵权法通常会和其他评价方法结合起来，比如和TOPSIS一起使用。

TOPSIS 法学习总结

TOPSIS 法主要是用来做综合评价和排序的。它的思想也比较好理解，就是先设定一个“最理想的对象”和一个“最差的对象”。最理想的对象就是每个指标都表现最好，最差的对象就是每个指标都表现最差。然后再看每个评价对象离最优对象有多近，离最差对象有多远。
如果一个对象离最优解越近、离最差解越远，就说明它综合表现越好，排名也就越靠前。TOPSIS 的步骤一般包括指标正向化、标准化、确定权重、构造加权矩阵、找出正理想解和负理想解，最后计算距离和综合得分。TOPSIS的优点是思路清楚，结果也比较直观，最后可以直接得到每个对象的分数和排名。它很适合用在多指标评价问题中，比如学校评价、比赛评价、城市发展水平评价等。TOPSIS的结果也会受到指标选择和权重设置的影响。如果一开始选的指标不合理，或者权重不合适，最后的排序结果也可能不太准确。

两者结合的理解
熵权法和 TOPSIS 经常放在一起使用。熵权法先根据数据算出每个指标的权重，TOPSIS 再利用这些权重计算综合得分并排序。
可以简单理解为：
熵权法负责算权重，TOPSIS 负责算得分和排名。这样结合起来，既能让权重确定更加客观，也能得到比较清晰的综合评价结果。


一、灰色关联分析

灰色关联分析是一种用来判断多个因素与目标结果之间关系密切程度的方法。它主要看各因素的变化趋势是否和目标结果相似，变化趋势越接近，说明关联度越高。

灰色关联分析常用于影响因素分析、指标重要性排序、综合评价和多方案比较。例如在评价比赛价值时，可以分析学校认可度、获奖难度、含金量、参赛成本等因素与综合评价之间的关系。

它的基本步骤是：先确定参考序列，也就是目标结果；再确定比较序列，也就是各个影响因素；然后对数据进行标准化处理，消除不同单位的影响；接着计算各因素和目标序列之间的差值；最后计算灰色关联度。关联度越大，说明该因素越重要。

灰色关联分析的优点是对数据量要求不高，计算比较简单，结果也比较容易解释，适合信息不完整的问题。缺点是它只能说明变化趋势相似，不能直接说明因果关系，而且结果会受到指标选择和标准化方法的影响。

二、线性规划

线性规划是一种优化方法，主要用来解决有限资源下的最优安排问题。它关注的是在满足各种约束条件的情况下，怎样使收益最大或者成本最小。

线性规划通常包括三个部分：决策变量、目标函数和约束条件。

决策变量是需要求解的未知量。例如 x1 表示蓝桥杯投入时间，x2 表示数学建模投入时间，x3 表示挑战杯投入时间。

目标函数表示想要达到的目标。例如要让收益最大，可以写成 Max Z = 8x1 + 9x2 + 7x3。

约束条件表示现实中的限制。例如总时间不能超过 100 小时，可以写成 x1 + x2 + x3 ≤ 100；同时还可以设置 x1 ≥ 10，x2 ≥ 20，x1、x2、x3 ≥ 0 等条件。

线性规划常用于资源分配、时间安排、成本最小化、收益最大化、生产计划和运输问题等场景。

它的优点是模型结构清晰，适合求最优方案，求解方法也比较成熟。缺点是要求目标函数和约束条件是线性的，而现实问题有时比较复杂，需要进行简化处理。

三、两者区别

灰色关联分析主要用于判断哪些因素更重要，偏向于分析问题。

线性规划主要用于求最优方案，偏向于解决问题。

简单来说，灰色关联分析是判断谁更重要，线性规划是决定怎么做最优。

四、学习总结

灰色关联分析和线性规划都是数学建模中常见的方法，但它们的作用不同。灰色关联分析适合用来分析影响因素，找出与目标结果关系更密切的指标。线性规划适合用来进行资源分配和方案优化，在有限条件下得到最优结果。

在实际建模中，可以先用灰色关联分析找出关键指标，再用线性规划进行优化决策。这样模型思路会更加完整，也更符合实际问题的分析过程。


整数规划学习总结

通过学习整数规划，我理解到它其实是线性规划的一种扩展。普通线性规划中，决策变量可以是小数，但在很多实际问题中，有些变量必须是整数，比如人数、车辆数、机器数量、是否选择某个项目等，这时候就需要用整数规划。

整数规划一般也包括决策变量、目标函数和约束条件。它和线性规划最大的区别就是：部分或全部决策变量必须取整数。如果变量只能取 0 或 1，那么这种问题也叫 0-1 整数规划。比如某个项目选不选、某条路线走不走、某个设备买不买，都可以用 0 和 1 表示。

我觉得整数规划最重要的是学会把现实问题中的“选择”和“数量”转化成变量。比如安排人员、分配任务、选择投资项目、规划路线等问题，很多都不能出现 2.5 个人或者 3.7 辆车，所以整数约束就很有必要。

整数规划的优点是更贴近现实，因为现实中很多决策本来就是离散的；但它的求解通常比普通线性规划更复杂，尤其是变量很多的时候，计算难度会明显增加。

总的来说，整数规划解决的就是：在变量必须是整数的情况下，怎样选择或安排才能达到最优。

非线性规划学习总结

通过学习非线性规划，我理解到它主要是用来解决目标函数或约束条件中含有非线性关系的最优化问题。和线性规划相比，非线性规划更适合描述一些复杂的实际情况。

在线性规划中，目标函数和约束条件都是一次关系，比如几个变量相加、相减。但在现实问题中，很多关系并不是简单线性的，可能会出现平方、乘积、指数、对数等形式。比如成本和产量之间可能不是简单成正比，收益增长到一定程度后可能会变慢，这些问题就更适合用非线性规划来描述。

非线性规划同样需要先确定决策变量、目标函数和约束条件。只不过它的建模过程往往更灵活，也更难一些。因为非线性问题可能存在多个局部最优解，不一定像线性规划那样容易找到全局最优解。

我觉得非线性规划的重点是理解实际问题中变量之间的关系。如果这些关系不是简单的线性关系，就要考虑是否需要建立非线性模型。它的优点是表达能力更强，更符合复杂现实问题；缺点是求解难度较大，对模型和算法的要求也更高。

总的来说，非线性规划解决的就是：在复杂非线性关系下，怎样找到最优方案。

两者的区别

整数规划更强调变量的取值限制，重点是变量必须是整数，适合处理“选不选”“要几个”“怎么分配”这类离散决策问题。

非线性规划更强调函数关系的复杂性，重点是目标函数或约束条件不是线性的，适合处理收益、成本、效率等存在复杂变化关系的问题。

可以简单理解为：

整数规划解决变量能不能取小数的问题，非线性规划解决关系是不是直线的问题。




图论与最短路径算法学习总结

通过学习图论，我理解到它主要是用来研究“点”和“边”之间关系的一种数学工具。很多实际问题都可以抽象成图，比如城市之间的道路、网络中的连接、人与人之间的关系、任务之间的依赖等。其中，点可以表示城市、地点、人物或任务，边可以表示道路、关系、距离或成本。

最短路径算法是图论中很常见的一类问题，主要解决的是：从一个点到另一个点，怎样走距离最短、花费最少或者时间最短。比如地图导航、物流配送、网络路由等问题，都可以用最短路径模型来分析。

常见的最短路径算法有 Dijkstra 算法、Floyd 算法和 Bellman-Ford 算法。Dijkstra 算法适合求一个点到其他点的最短距离，但一般要求边权不能为负数；Floyd 算法适合求任意两个点之间的最短路径，比较适合点数不多的情况；Bellman-Ford 算法可以处理负权边，但效率相对低一些。

我觉得学习最短路径算法时，最重要的是先把实际问题转化成图。也就是先确定哪些东西是点，哪些关系是边，边上的权值表示什么。只要图建立好了，后面就可以选择合适的算法来求解。

总的来说，图论与最短路径算法解决的就是：在一个关系网络中，怎样找到代价最小的路径。

网络最大流问题学习总结

通过学习网络最大流问题，我理解到它主要是研究在一个有方向、有容量限制的网络中，最多可以从起点输送多少“流量”到终点。这里的“流量”不一定是真正的水流，也可以表示车辆、货物、信息、资金、人员等。

最大流问题中一般有三个重要概念：源点、汇点和边的容量。源点就是流量的出发点，汇点就是流量最终到达的点，边的容量表示这条路径最多能通过多少流量。整个问题的目标就是在不超过每条边容量限制的情况下，让从源点到汇点的总流量最大。

常见的最大流算法有 Ford-Fulkerson 算法、Edmonds-Karp 算法和 Dinic 算法。它们的核心思想都是不断寻找可以继续增加流量的路径，也就是增广路，然后不断调整流量，直到无法再增加为止。

我觉得最大流问题比较适合用来解决资源分配和通道容量问题。比如交通疏导、物流运输、网络带宽分配、管道输送、任务匹配等，都可以抽象成最大流模型。它的难点主要在于建图，因为现实问题不一定一眼就能看出源点、汇点和容量限制。

总的来说，网络最大流问题解决的就是：在容量有限的网络中，最多能输送多少资源。

两者的区别

图论与最短路径算法更关注“怎么走最省”，重点是找到从一个点到另一个点的最短距离或最低成本路径。

网络最大流问题更关注“最多能运多少”，重点是在各种容量限制下，让整体通过的流量最大。

可以简单理解为：

最短路径看的是路线最优，最大流看的是总量最大。


最小费用最大流问题学习总结

通过学习最小费用最大流问题，我理解到它是在最大流问题的基础上，又加入了“费用”这个因素。最大流只关心从源点到汇点最多能输送多少流量，而最小费用最大流不仅要求流量尽可能大，还要求在达到最大流量的情况下，总费用尽可能小。

在这个问题中，每条边通常有两个属性：一个是容量，表示这条边最多能通过多少流量；另一个是费用，表示单位流量经过这条边需要付出的代价。整个问题的目标就是在满足容量限制的前提下，把尽可能多的流量送到终点，并且让运输成本最低。

我觉得这个模型比单纯的最大流更贴近现实。比如物流运输中，不仅要考虑最多能运输多少货物，还要考虑运输成本；人员分配中，不仅要安排尽可能多的人完成任务，还要考虑匹配成本或效率。它适合解决运输调度、资源分配、任务匹配、供应链优化等问题。

这个问题的难点主要还是建图。需要想清楚哪些是点，哪些是边，容量代表什么，费用又代表什么。只要模型建得合理，后面就可以用最小费用最大流算法来求解。

总的来说，最小费用最大流问题解决的就是：在尽可能多输送资源的同时，让总成本最低。

旅行商问题学习总结

通过学习旅行商问题，我理解到它是一个很经典的路径优化问题。它的基本意思是：一个旅行商需要从某个城市出发，经过所有城市一次，最后再回到起点，要求总路程最短。

这个问题看起来很简单，但实际上当城市数量变多以后，可能的路线数量会急剧增加，所以求解难度会变得很大。它不是简单地每次选择最近的城市就一定能得到最优解，因为局部最优不一定等于整体最优。

旅行商问题可以用来描述很多实际问题，比如物流配送路线规划、外卖配送、快递路径安排、生产加工顺序、机器人巡检路径等。只要问题中涉及“经过多个点，并且希望总代价最小”，就有可能和旅行商问题有关。

我觉得学习旅行商问题时，重点不是只记住算法，而是理解它为什么难。对于规模比较小的问题，可以用枚举、动态规划等方法求精确解；对于规模比较大的问题，通常会用贪心算法、遗传算法、模拟退火、蚁群算法等近似方法来求一个比较好的解。

总的来说，旅行商问题解决的就是：怎样用最短的路线走完所有指定地点。

插值算法学习总结

通过学习插值算法，我理解到它主要是用来根据已知数据点，估计未知位置上的数值。简单来说，就是我们已经知道一些点的数据，但中间有些位置没有数据，这时候就可以通过插值来补出中间的值。

插值算法的基本思想是：根据已有数据点之间的变化规律，构造一个函数或曲线，使它经过这些已知点，然后利用这个函数去估计其他位置的值。常见的插值方法有线性插值、拉格朗日插值、牛顿插值和样条插值等。

线性插值比较简单，就是认为两个点之间大致按直线变化；拉格朗日插值和牛顿插值可以构造多项式来拟合已知点；样条插值则更平滑，适合处理变化比较连续的数据。

我觉得插值算法在数据处理里很实用。比如实验数据缺失、曲线补全、图像处理、数值计算、预测中间值等，都可以用到插值。它的优点是方法直观，能根据已有数据补充未知数据；但缺点是如果数据本身波动很大，或者插值区间太远，结果可能不太可靠。

总的来说，插值算法解决的就是：根据已知数据，估计中间缺失或未知的数据。

三者的区别

最小费用最大流问题更偏向于网络优化，关注的是在容量限制下，怎样让流量最大并且费用最小。

旅行商问题更偏向于路径规划，关注的是怎样走完所有地点并让总路程最短。

插值算法更偏向于数据处理，关注的是怎样根据已有数据估计未知数据。

可以简单理解为：

最小费用最大流是“多运还要省钱”，旅行商问题是“走完所有点还要最短”，插值算法是“用已知数据补未知数据”。



拟合算法学习总结

通过学习拟合算法，我理解到它主要是用来根据已有数据，找出一个比较合适的函数关系。简单来说，就是我们有一组数据点，但不知道它们背后具体满足什么规律，这时候就可以用拟合算法找出一条曲线或一个模型，让它尽量接近这些数据。

拟合和插值不太一样。插值通常要求曲线经过所有已知点，而拟合不一定要求完全经过每个点，它更看重整体趋势。因为现实数据中经常会有误差或者噪声，如果强行经过所有点，反而可能不合理。

常见的拟合方法有线性拟合、多项式拟合、指数拟合、对数拟合等。比如数据大致呈直线变化，就可以用线性拟合；如果增长越来越快，可以考虑指数拟合；如果变化趋势比较弯曲，也可以考虑多项式拟合。

我觉得拟合算法在数学建模中很常用，尤其适合用来分析数据规律、建立预测模型、估计未来趋势。它的优点是直观、容易解释，也方便结合图像展示结果；但缺点是如果函数形式选得不合适，拟合出来的结果可能看起来很好，但实际预测效果不一定好。

总的来说，拟合算法解决的就是：根据已有数据，找到一个能描述整体变化趋势的函数模型。

微分方程学习总结

通过学习微分方程，我理解到它主要是用来描述某个量随时间或空间变化的规律。很多现实问题不是只看某一时刻的结果，而是要看它是怎样变化的，比如人口增长、疾病传播、物体运动、药物浓度变化、资源消耗等，这些都可以用微分方程来建模。

微分方程的核心思想是用“变化率”来描述系统。比如人口增长问题中，不是直接写人口是多少，而是写人口增长速度和当前人口之间的关系。通过这种方式，就可以描述一个系统从开始到后来的变化过程。

在数学建模中，常见的微分方程模型有指数增长模型、Logistic 阻滞增长模型、传染病模型、冷却模型等。它们都不是单纯地给出一个静态结果，而是更关注变量随时间变化的动态过程。

我觉得微分方程比较适合处理动态变化问题。它的优点是能够反映事物变化的过程，模型解释性比较强；但缺点是建模和求解相对难一些，有些方程不容易得到解析解，需要用数值方法来近似计算。

总的来说，微分方程解决的就是：描述一个系统是怎样随着时间或空间不断变化的。

两者的区别

拟合算法更偏向于从已有数据中找规律，重点是让模型尽量贴近数据。

微分方程更偏向于从变化机制出发建模，重点是描述变量之间的动态变化关系。

可以简单理解为：

拟合算法是“根据数据找趋势”，微分方程是“根据变化规律建模型”。





时间序列学习总结

通过学习时间序列，我理解到它主要是用来研究数据随时间变化规律的一种方法。简单来说，就是一组数据不是随便排列的，而是按照时间顺序记录下来的，比如每天的销量、每月的气温、每年的 GDP、股票价格变化等。

时间序列分析的重点是看数据有没有趋势、周期性和波动性。趋势指的是数据整体是上升、下降还是比较平稳；周期性指的是数据会不会按照一定时间规律重复变化；波动性指的是数据在不同时间点上的起伏情况。

在数学建模中，时间序列常常用来做预测。比如根据过去几个月的销售额预测下个月的销售额，根据前几年的用电量预测未来用电需求。常见的方法有移动平均法、指数平滑法、ARIMA 模型等。如果数据变化比较复杂，也可以结合机器学习方法进行预测。

我觉得时间序列的关键不是只会套模型，而是先观察数据本身的变化特点。比如数据有没有明显趋势，是否存在季节性，是否有异常点。如果不先分析数据特点，直接建模，预测结果可能就不太可靠。

时间序列的优点是很适合处理和时间有关的预测问题，结果也比较容易解释；但它也有缺点，比如对历史数据依赖比较强，如果未来环境发生明显变化，单纯依靠过去数据预测就可能不准确。

总的来说，时间序列解决的就是：根据过去一段时间的数据变化规律，分析趋势并预测未来。

使用场景

时间序列适合用在和“时间变化”有关的问题中。比如预测未来销量、气温变化、人口增长、交通流量、用电需求、股票价格、疫情发展趋势等。

在数学建模比赛中，如果题目给的是按年份、月份、天数或小时排列的数据，并且要求分析未来变化趋势，就可以考虑使用时间序列模型。

可以简单理解为：

只要数据是按时间顺序变化的，并且需要预测后面的走势，就可以考虑时间序列。
