# Gate 0：题目与材料诊断

进入模型路线选择前，必须同时完成题目理解和材料完整性确认。

## 机器校验（必须通过）

- 已执行 `python scripts/verify_materials.py --materials <材料目录> --problem <题号>`。
- `material_manifest.json` 存在且符合 `schemas/material_manifest.schema.json`。
- 官方题面、必需附件、必需模板均已声明并存在。
- 所有已声明文件的 SHA-256 与清单一致。
- 清单未声明包含答案、题解或参考结论。

任何一项失败，运行状态必须为 `blocked`；不得仅因材料目录存在而标记 `ready`。

## 人工诊断

### 一、题目理解

- 是否复述了题目背景？
- 是否明确了每个问题要解决什么？
- 是否区分了已知条件、未知量、要求输出？
- 是否识别了硬约束、隐含约束和提交格式要求？

### 二、子问题拆解

- 是否只把题面明确编号的问题作为子问题？
- 是否说明了子问题之间的依赖关系？
- 是否指出前一问结果如何服务后一问？

### 三、题型判断与数据需求

- 是否判断每一问属于优化、预测、评价、分类、仿真、机理建模、图论或规划？
- 已有数据、缺失数据和单位是否清楚？
- 是否识别异常值、缺失值、时间粒度或空间坐标风险？

## 输出要求

必须同时产出：

1. `diagnosis.md`：题目理解、子问题拆解、数据需求、候选模型和最大风险；
2. `diagnosis.json`：符合 `schemas/diagnosis.schema.json` 的同一诊断；
3. `material_review.json`：机器校验报告和人工 T0-T4、M1-M5 审查结论；
4. 人工确认项。

未通过 Gate 0，不允许进入 Gate 1。
