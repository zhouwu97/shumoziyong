# Gate 0：总控诊断闸门

进入建模前，必须完成以下检查。

## 一、题目理解

- 是否复述了题目背景？
- 是否明确了每个问题要解决什么？
- 是否区分了已知条件、未知量、要求输出？
- 是否识别了题目里的硬约束、隐含约束和提交格式要求？

## 二、子问题拆解

- 是否只把题面明确编号的问题作为子问题？
- 是否说明了子问题之间的依赖关系？
- 是否说明前一问结果如何服务后一问？
- 是否指出哪些子问题可以并行推进，哪些必须等待前置结果？

## 三、题型判断

- 本题属于优化、预测、评价、分类、仿真、机理建模、图论、规划中的哪几类？
- 每一问分别是什么数学任务？
- 是否存在混合题型？
- 是否有明显不能使用的模型路线？

## 四、数据需求

- 已有数据有哪些？
- 缺失数据有哪些？
- 哪些变量需要从附件计算？
- 哪些变量需要假设？
- 数据单位是否统一？
- 是否有异常值、缺失值、时间粒度或空间坐标问题？

## 五、模型候选

- 至少给出 2 种可行模型路线。
- 必须比较优缺点。
- 必须说明为什么暂时不选更复杂模型。
- 必须指出每条路线的实现难度和失败降级方案。

## 六、输出要求

必须输出：

1. 总控诊断表；
2. 子问题任务表；
3. 数据需求表；
4. 候选模型比较表；
5. 人工确认项；
6. 最大风险列表。

未通过本闸门，不允许进入建模、代码和论文阶段。
