# Gate 2：代码前闸门

进入代码和图表阶段前，必须完成以下检查。

## 输入与输出

- 每个代码模块的输入文件是否明确？
- 每个代码模块的输出文件是否明确？
- 中间结果是否需要保存到 `results/`？
- 图表是否需要保存到 `figures/`？

## 模型固化

- 代码是否只实现已经确认的模型？
- 是否禁止在代码阶段临时更换模型路线？
- 参数、约束和评价函数是否已经在建模报告中写清？
- 是否有基线模型用于验证结果合理性？

## 验证方式

- 是否设计了单元级检查或结果 sanity check？
- 是否检查约束满足情况？
- 是否检查异常值和边界条件？
- 随机算法是否设置随机种子和重复运行方案？

## 输出要求

代码阶段必须生成：

1. 可复现的代码文件；
2. `results/` 下的结果表或中间数据；
3. `figures/` 下的图表；
4. `reports/RESULTS_REPORT.md`；
5. 每个结果对应的数据来源、代码来源和模型步骤说明。

未通过本闸门，不允许进入论文写作阶段。
