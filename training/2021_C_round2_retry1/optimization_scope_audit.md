# 2021-C 优化结论作用域审计

## 1. 审计结论

本次审计先确认旧实现的作用域，再按执行计划允许的最小分支完成定向修复。结论枚举如下：

```text
修复前：candidate_set_optimum_only
修复前：pressure_test_scope_overstated
修复前：claim_wording_overstated
修复后：globality_confirmed
```

旧实现先按净产品能力固定具体供应商集合，再在该集合内优化成本和损耗，因此旧成本、损耗只能称为候选集内最优。旧压力测试也固定具体候选集合，使“需求增加 10%不可行”和“关键供应商全期中断不可行”的论文表述超过了代码证据范围。

修复后的 Python 与 MATLAB 问题二模型均保留全部 402 家正能力供应商，并按“最小供应商基数、最小采购成本、最小运输损耗”三阶段顺序求解。第二阶段只锁定最优基数值 26，第三阶段只锁定最优成本值，不固定具体名单或完整前序解。

## 2. 六项定点审计

| 审计项 | 修复前 | 修复后 | 证据与结论 |
|---|---|---|---|
| 1. 第一阶段变量是否覆盖全部 402 家 | 否。能力排序只用于选择固定集合 | 是。402 家均进入正式 MILP | `raw_solution.json` 中供应商数与候选数均为 402；单周含 3,216 个运输流变量和 3,216 个二元指派变量 |
| 2. 第二阶段固定数量还是名单 | 固定具体名单 | 只锁定 `sum z_i = 26` | 26 家组合在全部候选中继续自由选择 |
| 3. 第三阶段固定成本值还是完整解 | 在固定名单上锁定成本值 | 只锁定全空间第二阶段成本值 | 未锁定具体供应商名单、运输分配或完整第二阶段解 |
| 4. 成本与损耗目标对应哪个空间 | 固定候选集 | 全部 402 家中基数为 26 的可行空间 | Python 三阶段均为 `Optimal`、`mip_gap=0`；MATLAB 三阶段 `exitflag=1` |
| 5. 压力测试是否重新开放全部供应商和数量 | 否 | 是 | 每个情景重新构造完整候选模型；供应商数量随情景重新优化 |
| 6. “最优、最少、不可行”措辞是否一致 | 否 | 是 | 论文改为明确完整决策空间与求解状态；限时可行不再写成不可行 |

## 3. 最少供应商数证明

用最低历史平均损耗构造净产品能力上界，并将 402 家供应商按该上界降序排列。能力最强的前 25 家合计只能覆盖 28,044.405804 m³/周，小于基准需求 28,200 m³/周，因此任何 25 家组合都不可能满足需求。完整 MILP 在 26 家时返回可行且最优的解，因此基准模型的最少供应商数为 26。

该证明只说明当前确定性参数和已建模约束下的最小基数，不等于企业应长期只维护 26 家合作供应商。

## 4. 三阶段词典序锁定

1. 在全部 402 家中最小化供应商基数，得到 `K*=26`。
2. 仅加入 `sum z_i = K*`，在所有 26 家组合中最小化相对采购成本。
3. 保留基数约束，并加入 `cost <= cost* + 1e-6`，在全部成本最优解中最小化运输损耗。

Python 三阶段的 `incumbent`、`best_bound` 和 `mip_gap` 分别记录在 `results/solver_status.json`。三阶段均证明最优，正式周相对采购成本为 20,463.567522，周运输损耗为 102.938185 m³。MATLAB 独立复现的 24 周采购成本与 Python 相差约 `2.33e-7`，所有连续量最大绝对误差约 `3.21e-7`，低于 `1e-6` 验收阈值。

## 5. 压力测试作用域

| 情景 | 完整模型结果 | 允许措辞 |
|---|---|---|
| 需求 -10% | 21 家，全局最优可行 | 完整模型可行 |
| 基准需求 | 26 家，全局最优可行 | 完整模型可行 |
| 需求 +10% | 找到可行 incumbent；60 秒内未证明采购成本最优 | `feasible_not_proven_optimal` |
| 损耗率乘 0.8 | 26 家，全局最优可行 | 完整模型可行 |
| 损耗率乘 1.2 | 26 家，全局最优可行 | 完整模型可行 |
| 关键供应商能力 -10% | 26 家，全局最优可行 | 完整模型可行 |
| 关键供应商全期中断 | 27 家，全局最优可行 | 完整模型可行 |

因此，旧论文中的“需求增加 10%不可行”和“关键供应商全期中断不可行”均被完整模型结果推翻。需求增加 10%只能报告已找到可行解但尚未证明成本最优；关键供应商中断情景应报告通过启用替代供应商恢复可行，不能再描述为全局不可行。

## 6. 参数语义简表

| 参数 | 统计含义 | 模型用途 | 论文允许名称 |
|---|---|---|---|
| 供货能力估计 | 非零供货均值乘按单供货概率形成的历史条件期望 | 确定性计划容量 | 历史期望供货能力、计划能力估计 |
| 损耗率 | 各转运商非零运输记录的历史均值 | 确定性运输损耗参数 | 历史平均损耗率、预测损耗率 |
| 最大周产能 | 当前确定性网络模型的最优目标值 | 网络能力上界估计 | 模型预测最大周产能、确定性模型产能上界 |

上述参数均不是未来保证量。论文不得将历史期望供货能力写成承诺产能，也不得将确定性模型上界写成企业长期必然实现的产量。
