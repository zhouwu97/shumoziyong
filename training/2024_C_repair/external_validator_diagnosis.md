# 冻结外部 Validator 黑盒诊断（封存阻断）

## 执行边界

按用户提供的 `validators.problem_positive_v2.validate.validate_result()` 接口执行了纯数学黑盒复核：输入为官方附件 1、附件 2 与本训练导出的 `results/formal_result.json`。未读取 `validators/` 下的源码，未运行 `validate_a092_formal_run.py`，未改动 A092 协议或运行目录。

## 导出合同确认

黑盒先后要求并接受了以下结构映射：

1. 顶层 `scenarios` 必须为列表；
2. 每项需有 `scenario_id`，分别为 `q1_waste`、`q1_discount`、`q2_frozen`、`q3_frozen`；
3. 每项含 `assignments` 和 `objective_reported`；
4. assignment 面积字段名为 `area_mu`。

最终导出文件为 `results/formal_result.json`，最终黑盒报告为 `results/frozen_external_validator_report.json`。因此失败不是 JSON 容器、场景命名或面积字段映射错误。

## 黑盒复核结果

最终报告 `valid=false`，四个场景均未通过：

- **Q1 滞销**：外部重算目标为 38,521,336.50 元，内部报告为 38,477,236.50 元，差 44,100.00 元；同时出现多项连续种植违约。
- **Q1 半价**：目标值一致，但出现多项连续种植违约。
- **Q2 冻结**：外部重算目标为 32,615,925.60 元，内部报告为 29,437,942.84 元，差 3,177,982.77 元；同时出现多项连续种植违约。
- **Q3 冻结**：外部重算目标为 32,406,619.21 元，内部报告为 29,136,973.13 元，差 3,269,646.07 元；同时出现多项连续种植违约。

## 根因判定

### 1. 连续种植口径冲突

外部报告在水浇地和普通大棚中，将同一作物在同一季次的跨年重复标为 `continuous_crop`，即使本轮的“实际有效季序列”中间已有另一季作物。这与 `assumptions.md` 的 A1 主口径实质不同；它不仅是数值容差，而会改变可行域和最终策略。

### 2. 销售上限聚合口径冲突

Q1 滞销的 44,100 元差异表明外部销售计算并非本轮“作物×地块类型×季次”基线口径。用户提供的外部 Validator 说明为“按作物—季次聚合销售上限”，这与本轮 A4 的地块类型保留口径不同。Q1 半价不显差异不能证明两者一致，因为该情景的超额部分仍有收入。

### 3. Q2/Q3 冻结参数合同不可形成

Q2/Q3 的外部重算目标与本轮保存的保守分位随机参数存在百万量级差异。黑盒接口只返回结果，不提供可用于建模的冻结参数、情景生成器或聚合规则。按“不得读取冻结外部 Validator 源码来迎合测试”的限制，无法从源码反推出该合同，也不能把外部重算值手工写回 `objective_reported` 伪造通过。

## 结论与停止点

这属于“Validator 数据合同无法成立”以及“两个实质影响结论的题意/合同解释冲突”的硬停止条件。当前训练只能表述为：**内部修复链条通过，冻结外部复核失败，未封存、未提交、不能作为 repair_passed。**

在获得冻结 Validator 的公开参数合同，或由项目维护者明确裁决 A1/A4 与 Q2/Q3 冻结规则前，禁止：放宽容差、手工修改 `objective_reported`、把内部报告替换为外部报告、commit、push 或进入盲测。
