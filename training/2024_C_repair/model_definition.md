# 2024-C 模型定义（Gate 1）

## 1. 集合、索引与数据

- `P`：54 个地块；每个地块有类型 `l(p)` 与面积 `A_p`。
- `T={2024,...,2030}`：规划年；`T0={2023,...,2030}`：含历史的年份。
- `C={1,...,41}`：作物编号；`B={1,...,5,17,...,19}`：豆类集合。
- `S_p`：地块 `p` 的可用季次集合；水浇地通过模式变量选择水稻单季或两季蔬菜。
- `G=(c,l,s)`：附件 2 的作物--地块类型--种植季次统计组。对每个 `g`，2023 销量基线 `D_g^0` 由附件 2 的同组实际面积与亩产量计算。
- `H_{p,c,s}`：附件 2 的 2023 历史种植面积；`Y_{c,l,s}`、`K_{c,l,s}`、`P_{c,l,s}` 分别为亩产量、亩成本和中点销售单价。

附件未列出智慧大棚第一季参数时，按官方注释继承普通大棚第一季的 `Y,K,P`，但保持独立统计组 `G`。

## 2. 决策变量

- `x_{p,c,t,s} >= 0`：地块 `p` 在年份 `t`、季次 `s` 种植作物 `c` 的面积（亩，连续变量）。
- `z_{p,c,t,s} in {0,1}`：作物 `c` 是否在该地块、该有效季有正面积，用于重茬和二级管理偏好。
- `m_{p,t} in {0,1}`：水浇地的模式；`m=1` 表示单季水稻，`m=0` 表示两季蔬菜。
- `q_{g,t} >= 0`：统计组 `g` 在年份 `t` 的正常销售产量（斤）。

面积变量不是整数。二元变量只表达活动状态、模式和分段销售的逻辑；所有导出前保留原始双精度值。

## 3. 目标函数

令 `R_{g,t}=sum_{p:l(p)=l} Y_{c,l,s} x_{p,c,t,s}` 为该统计组总产量，`D_{g,t}` 为相应需求上限，`alpha` 为超额部分销售比例（Q1-1 为 0，Q1-2 为 0.5，Q2/Q3 为 0）。

`0 <= q_{g,t} <= R_{g,t}`，且 `q_{g,t} <= D_{g,t}`。正式目标为：

`max F = sum_{g,t} [P_{g,t} q_{g,t} + alpha P_{g,t}(R_{g,t}-q_{g,t}) - sum_{p in g} K_{g,t} x_{p,c,t,s}] - epsilon sum z_{p,c,t,s}`。

其中 `epsilon=0.01` 元/活动位置，仅作收益相同情况下的二级排序。最终报告值不直接使用求解器目标，而由 `recompute_objective()` 对导出的 `x`、参数和销售规则独立计算。

Q1 使用 2023 参数及固定 `D_g^0`；Q2 使用 `assumptions.md` 中 Q2 情景的保守分位参数；Q3 使用相关情景的保守分位参数。模型提供可验证的可行策略，因混合整数求解的时间上限和情景近似，不声明全球最优。

## 4. 硬约束、实现计划与验证计划

| ID | 数学表达/口径 | 代码实现计划 | 独立验证计划 |
| --- | --- | --- | --- |
| C01 容量 | `sum_c x_{p,c,t,s} <= A_p`，仅对实际允许季次；单季地块每年仅单季 | `build_model.py` 添加逐地块季容量行 | `check_land_capacity()` 报告每个超额位置 |
| C02 适种性与水浇地模式 | 不兼容组合 `x=0`；水浇地 `x_rice <= A_p m`，`x_veg <= A_p(1-m)`，`m in {0,1}` | 由附件 1 生成允许组合；水浇地加模式变量 | `check_crop_land_compatibility()` 与 `check_water_mode()` |
| C03 重茬 | 对每一对相邻有效季和同一作物，`z_i+z_j<=1`；`x_i<=A_p z_i` | 从地块类型生成有效季邻接表，含历史前驱 | `check_continuous_crop()`、`check_2023_to_2024_boundary()` |
| C04 豆类窗口 | 对每个 `p,w`：`sum_{t in w,c in B,s} x_{p,c,t,s}+sum_{c in B,s}H_{p,c,s}[2023 in w] >= A_p` | 建立 6 个完整滚动窗口的下界行 | `check_three_year_legume_windows()` 输出地块--窗口缺口 |
| C05 销售与收益 | `q<=R`、`q<=D`；正式收入按 Q1/Q2/Q3 对超额部分的规则计算 | 为每组/年建立 `q` 和两类不等式 | `recompute_objective()` 重算产量、正常/超额销量与收入；`check_sales_limit()` 核对索引 |
| C06 非负与离散性 | `x,q>=0`；`z,m in {0,1}` | 变量下界与 integrality 向量 | `check_integrality_and_nonnegative()`；检查原始 JSON 未经展示舍入 |
| C07 管理偏好 | 非硬约束；仅 `epsilon sum z` 二级惩罚 | 目标二级项 | 仅统计活动位置数，不计入硬违约 |
| C08 随机复现 | 参数由固定种子及可保存算法产生 | `scenario_generation.py` 统一生成、写入 CSV/JSON | `check_random_reproducibility()` 重新生成并逐项比对 |

## 5. 算法与可执行边界

采用 SciPy/HiGHS 的混合整数线性规划求解器。可行域由附件直接构造；连续面积由线性变量表达，活动/模式逻辑由二元变量表达，销售分段收入以辅助变量线性化。求解器最多运行 300 秒/场景；若达到时间限但返回可行解，将在结果中标记为 `feasible_time_limited`，不作为最优性声明。

Q2/Q3 不是把 128 个情景复制到一个巨大确定性等价模型，而是以保存的样本生成保守分位参数后求解，并以全样本和独立样本复核风险。这是为保证训练在可复现、可审核时间内完成的样本稳健近似；其适用范围和局限性将在论文及审核中明确。
