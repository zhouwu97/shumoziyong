# patch_B311_spatial_coverage_optimization

- 类型：论文经验补丁草案
- 来源论文：2023 年高教社杯 B311《基于主要目标法的测线设计问题》
- 适用题型：空间覆盖优化 / 工程测线布设 / 几何建模 / 条带扫描 / 多目标规划
- 依赖：`prompt_base_v1.0.md`
- 进入 base：否

## 职责边界

本补丁只记录 B311 的可迁移经验和误用风险，不重复 base 的通用规则，不负责旧题闭环，不负责 stable 判定，不把单篇论文经验提升为通用规则。

## 核心经验

```text
先定义覆盖单元和重叠口径
-> 再寻找覆盖宽度稳定的方向
-> 最后用边界贴合和临界约束递推布线
```

B311 的价值不在于某个具体公式，而在于它把“测线怎么铺”拆成了覆盖宽度计算、重叠率定义、方向选择和测线递推四个可检查环节。

## 可迁移经验

1. 空间覆盖题先识别覆盖单元：条带、视场、服务半径、扫描宽度、作用范围或影响区。
2. 明确覆盖评价口径：覆盖率、漏覆盖率、重叠率、边界覆盖、覆盖冗余和总路径长度。
3. 指标定义有歧义时，先建立候选定义，用图示、极端情形、临界边界或现实含义比较后再选用。
4. 复杂空间几何题可先在二维剖面中推导，再通过方向角、投影、线面角或坐标变换推广到三维。
5. 测线、路径或扫描线方向应优先选择能让覆盖宽度或服务范围稳定的方向。
6. 确定方向后，可用“第一条线贴边，后续线按最小可接受重叠率递推”的方式构造候选方案。
7. 递推结束后必须做边界验算，确认最后一条线是否真正覆盖终止边界。
8. 不规则地形或非均匀数据可先分区，再在每个分区内使用相对规则的覆盖模型。
9. 多目标工程题若存在明确主优先级，可采用主要目标法，将其他目标转为约束、惩罚项或验算指标。
10. 结果论证应同时包含几何示意图、指标表、空间布设图、边界检查和局限说明。

## 适用条件

遇到以下题目时可以启用本补丁：

1. 题目涉及测线、路线、扫描线、巡检线、航迹或覆盖带布设；
2. 覆盖宽度、作用半径或服务范围随位置、方向、深度、高度、坡度或地形变化；
3. 需要同时考虑覆盖率、漏覆盖、重叠冗余、总长度或总成本；
4. 题目存在明显的空间方向选择，如等深线、等高线、主风向、道路方向、网格主轴或梯度方向；
5. 多目标之间有明显优先级，例如先保证安全或覆盖，再减少成本；
6. 结果需要用图形展示空间布设是否合理。

## 使用步骤

```text
1. 定义覆盖单元：覆盖带/视场/服务半径/影响区是什么？
2. 定义评价口径：覆盖、漏覆盖、重叠和总长度分别怎么计算？
3. 建立局部几何模型：覆盖宽度或作用范围由哪些变量决定？
4. 检查定义歧义：是否存在多个合理口径？是否需要候选模型比较？
5. 寻找稳定方向：哪个方向能让覆盖宽度、服务半径或约束最稳定？
6. 构造布设策略：第一条线如何贴边？相邻线如何按临界约束递推？
7. 做边界验算：起点、终点、角落、分区边界是否漏覆盖？
8. 做结果论证：表格给指标，图像看布设，文字解释取舍和风险。
```

## 误用风险

1. 不要把“沿等深线布设”直接迁移到所有路径规划题；应先证明该方向能稳定覆盖宽度或降低约束复杂度。
2. 不要把“最小重叠率”当作所有题的最优策略；存在测量误差、安全裕度和不确定性时应保留冗余。
3. 不要把主要目标法等同于多目标最优；它是工程取舍方法，需要说明被降级目标的处理方式。
4. 不要只用递推结果证明最优；递推方案通常是可解释构造，还需要边界检验、对照方案或敏感性分析支撑。
5. 不要用单篇论文的具体公式、数值、分区和代码替代新题建模。

## 不可照搬内容

以下内容只适用于 B311 或多波束测深背景，不能直接迁移：

1. 多波束测深覆盖宽度公式；
2. B311 的开角、坡度、水深、测线间距和海域尺寸；
3. B311 的 34 条测线、68 海里总长度、466394.4m 总长度和 2.3% 漏测率；
4. 依据等高线将问题四划分为三块测区的具体方案；
5. MATLAB/Python 代码结构；
6. 对《海道测量规范》(GB12327-2022)的具体引用结论。
