# patch_A127_engineering_layout_optimization

- 类型：论文经验补丁
- 来源论文：2023 年高教社杯 A127《定日镜场的输出功率优化》
- 适用范围：工程优化 / 空间布局 / 阵列规划 / 光伏光热 / 遮挡仿真优化
- 进入 base：否

## 职责边界

本补丁只记录 A127 的空间布局和工程优化经验，不重复 base/plugin 的通用规则，不负责旧题闭环或 stable 判定。

## 触发条件

当题目同时满足以下特征时启用本补丁：

1. 存在大量同类对象的位置、尺寸、高度、角度或排布变量；
2. 目标函数与工程效率、输出功率、覆盖率、利用率或成本收益有关；
3. 约束包含空间范围、对象间距、尺寸上下限、额定指标或安全距离；
4. 直接逐对象优化会导致变量维度过高；
5. 结果需要给出可执行的工程设计参数。

## 可迁移经验

1. 空间布局题要先识别对象、坐标、姿态、损失项、单对象输出和系统总输出。
2. 大规模同类对象不宜直接逐对象优化，优先考虑分区、分层、规则化或参数化。
3. 可先构造基础可行布局，再做粗搜索、细搜索和局部精修。
4. 规则化布局只能作为可解释的降维方案，不能声称天然最优。
5. 搜索过程需要给出可行性检查、停止条件、对照方案和敏感性分析。
6. 如果某个局部指标下降，必须解释它换来了什么整体收益。

## 推荐建模路线

```text
工程机制拆解
-> 评价函数/目标函数
-> 决策变量和约束
-> 变量降维或规则化
-> 基础可行方案
-> 粗搜索
-> 细搜索
-> 结果解释和敏感性分析
```

如果存在复杂物理机制，可按以下链路拆解：

```text
对象坐标
-> 对象姿态/状态
-> 损失项或效率项
-> 单对象输出
-> 系统总输出
```

## 降维策略

可优先尝试以下方式：

1. 分区：按距离、方向、功能区、风险区或资源密度划分；
2. 分层：按半径、等级、时间段、负荷等级或优先级划分；
3. 规则化：网格、队列、分组、对称布局、等间距布局；
4. 参数化：用少量参数控制大量对象的位置或规格；
5. 固定结构后优化参数：先确定布局结构，再优化尺寸、高度、数量等参数。

输出时需要说明降维减少了哪些变量、可能排除了哪些更优方案，以及如何用对比或敏感性分析降低风险。
