# patch_A092_engineering_optimization

- 类型：论文经验补丁
- 来源论文：2023 年高教社杯 A092《定日镜场的优化设计》
- 适用题型：工程优化 / 机理建模 / 仿真计算 / 布局优化
- 依赖：`prompt_base_v1.0.md` + `plugin_optimization_v1.md`
- 进入 base：否

## A092 最值得迁移的经验

```text
先建立基础评价模型
-> 再把后续问题转化为约束优化
```

A092 的价值不在于遗传算法本身，而在于：

1. 先拆解工程机制；
2. 建立光学效率和输出功率评价模型；
3. 把后续设计问题转化为约束优化；
4. 变量过多时利用结构规律降维；
5. 用图表和优化前后对比解释结果。

## 适用条件

遇到以下题目时启用本补丁：

1. 工程系统效率优化；
2. 物理机制或能量流明显；
3. 布局、位置、尺寸、高度、数量等变量需要优化；
4. 输出结果需要满足约束并尽量提高某项性能；
5. 变量维度高，直接优化困难；
6. 需要图表展示空间分布、趋势或优化效果。

## A092 工程优化补丁提示词

```text
你现在叠加 A092 工程优化类经验补丁。

请不要直接套用 A092 的太阳能公式、定日镜公式、同心圆布局或蒙特卡洛截断效率。你要迁移的是它的建模路线，而不是题目细节。

一、机制拆解先行
如果题目包含复杂工程或物理机制，必须先拆解：
1. 系统主体；
2. 输入量；
3. 输出量；
4. 损失项；
5. 效率项；
6. 约束来源；
7. 可控变量；
8. 不可控环境变量。

二、评价函数先行
优化前必须先回答：
1. 如何评价一个方案好坏；
2. 评价指标是否可计算；
3. 评价函数输入是什么；
4. 评价函数输出是什么；
5. 评价函数如何服务后续优化；
6. 如果没有评价函数，为什么不能直接优化。

三、从评价到优化
请按以下链路设计模型：
机制变量
-> 指标分解
-> 基础评价函数
-> 目标函数
-> 决策变量
-> 约束条件
-> 求解算法
-> 结果解释

四、高维变量处理
如果变量过多，必须考虑：
1. 是否存在空间分区；
2. 是否存在时间分段；
3. 是否可以把同类对象合并；
4. 是否可以按层、组、区域设置相同参数；
5. 是否可以用规则生成变量；
6. 是否可以先做统一参数优化，再做非统一扩展。

五、算法选择约束
如果选择遗传算法、粒子群、模拟退火等智能算法，必须说明：
1. 基础评价函数已经建立；
2. 目标函数已经明确；
3. 决策变量和约束已经明确；
4. 为什么简单遍历不可行；
5. 为什么规划方法不适合；
6. 如何设置算法参数；
7. 如何重复运行验证稳定性；
8. 是否只能称为近似最优。

六、结果论证必须包含
1. 核心结果表；
2. 趋势图；
3. 空间分布图或结构图；
4. 优化前后对比；
5. 关键变量解释；
6. 敏感性分析；
7. 约束满足情况；
8. 异常结果解释。
```

## 可迁移表达模板

```text
针对该优化问题，本文首先建立用于衡量方案优劣的评价函数，而不是直接选择优化算法。该评价函数以……为输入，以……为输出，能够刻画……的综合效果。
```

```text
在评价函数明确后，本文将原问题转化为以……最大化/最小化为目标、以……为决策变量、以……为约束条件的优化问题。
```

```text
由于直接对所有变量进行搜索会导致维度过高，本文根据……规律对变量进行分区/分层/参数化处理，从而在保留主要结构特征的同时降低求解复杂度。
```

## 禁止照搬内容

以下内容不能从 A092 直接照搬：

1. 太阳高度角和方位角公式；
2. 定日镜光学效率公式；
3. 同心圆布局假设；
4. 蒙特卡洛截断效率；
5. MATLAB 代码结构；
6. A092 的具体数值和图表。

这些只在同类太阳能定日镜问题中适用。
