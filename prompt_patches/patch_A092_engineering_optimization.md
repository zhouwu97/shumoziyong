# patch_A092_engineering_optimization

- 类型：论文经验补丁
- 来源论文：2023 年高教社杯 A092《定日镜场的优化设计》
- 适用题型：工程优化 / 机理建模 / 仿真计算 / 布局优化
- 依赖：`prompt_base_v1.0.md` + `plugin_optimization_v1.md`
- 进入 base：否

## 职责边界

本补丁只记录 A092 的可迁移经验和误用风险，不重复 base/plugin 的通用规则，不负责旧题闭环或 `competition_evidenced` 判定。

## 核心经验

```text
先建立基础评价模型
-> 再把后续问题转化为约束优化
```

A092 的价值不在于某个优化算法，而在于它把复杂工程系统先拆成可计算评价模型，再把设计问题转化为目标函数、决策变量和约束清楚的优化问题。

## 可迁移经验

1. 复杂工程题先拆机制：主体、输入、输出、损失项、效率项、约束来源、可控变量和环境变量。
2. 优化前先定义方案好坏的评价函数，明确评价函数的输入、输出和现实意义。
3. 将评价函数作为后续目标函数或约束的一部分，避免直接从题面跳到算法。
4. 变量过多时，用分区、分层、规则化或参数化降低维度。
5. 结果论证要同时包含核心结果表、趋势解释、空间分布或结构图、优化前后对比、敏感性分析和约束满足情况。

## 适用条件

遇到以下题目时可以启用本补丁：

1. 工程系统效率优化；
2. 物理机制或能量流明显；
3. 布局、位置、尺寸、高度、数量等变量需要优化；
4. 输出结果需要满足约束并提高某项性能；
5. 变量维度高，直接逐变量优化困难；
6. 需要用图表展示空间分布、趋势或优化效果。

## 可迁移表达模板

```text
针对该优化问题，本文首先建立用于衡量方案优劣的评价函数，而不是直接选择优化算法。该评价函数以……为输入，以……为输出，能够刻画……的综合效果。
```

```text
在评价函数明确后，本文将原问题转化为以……最大化/最小化为目标、以……为决策变量、以……为约束条件的优化问题。
```

```text
由于直接对所有变量进行搜索会导致维度过高，本文根据……规律对变量进行分区/分层/参数化处理，从而在保留主要结构特征的同时降低求解复杂度。
```

## 不可照搬内容

以下内容只适用于同类太阳能定日镜问题，不能直接迁移：

1. 太阳高度角和方位角公式；
2. 定日镜光学效率公式；
3. 同心圆布局假设；
4. 蒙特卡洛截断效率；
5. MATLAB 代码结构；
6. A092 的具体数值和图表。
