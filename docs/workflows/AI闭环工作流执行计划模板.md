# AI 闭环工作流执行计划模板

## 1. 用途

本文件用于让 AI 在执行旧题闭环测试前，先自动写一份短计划，再按计划执行。

目的：

1. 防止主工作流过长导致 AI 忘记关键规则；
2. 防止 AI 跳过材料资格审查；
3. 防止 AI 把 T0/T1/T2 误计入 stable；
4. 防止 AI 直接写论文、写代码或给最终答案；
5. 防止 AI 在本地没有 T3 时直接停下来要人工选题；
6. 让每轮闭环都有清晰的执行待办和自检清单。

本文件不是替代 `docs/workflows/AI自动化闭环测试工作流.md`，而是执行前的压缩计划模板。

## 2. 执行原则

AI 每次开始闭环测试前，必须先做两件事：

```text
第一步：读取主工作流文档
第二步：根据本模板生成本轮执行计划
```

没有执行计划，不允许直接进入旧题诊断。

## 3. 必读文件

AI 必须按顺序读取：

```text
1. docs/workflows/AI自动化闭环测试工作流.md
2. docs/workflows/数模AI训练总控工作流v3.md
3. training_log.md
4. prompt_base/prompt_base_v1.0.md
5. prompt_plugins/plugin_optimization_v1.md
6. 本轮需要的 prompt_patches/
7. 本轮需要的 papers/
8. tests/old_problems/
9. reviews/failure_cards/
```

如果目标是自动选题，还必须扫描：

```text
高教社杯全国大学生数学建模竞赛优秀论文/
output/
tmp/
项目根目录下的 PDF、DOCX、XLSX、CSV、TXT、MD
```

## 4. 计划生成提示词

下面这段可以直接交给 AI。

```text
你现在是“数模 AI 闭环测试执行代理”。

在执行任何旧题诊断前，你必须先根据项目中的《docs/workflows/AI自动化闭环测试工作流.md》和《docs/workflows/AI闭环工作流执行计划模板.md》生成一份本轮执行计划。

注意：
你不是来写论文的。
你不是来写代码的。
你不是来求最终答案的。
你是来完成旧题总控诊断闭环测试的。

请先输出《本轮闭环执行计划》，必须包含：

1. 本轮目标
   - 是流程冒烟测试、材料受限测试、T3 泛化测试，还是 stable 候选测试；
   - 如果用户没有指定旧题，你要自动扫描候选题并选择最高资格题；
   - 如果本地没有 T3，你必须先自动尝试获取官方题面、附件和模板；
   - 官方材料获取失败后，才允许跑 T0/T1/T2 流程测试，并生成 T3 材料缺口清单。

2. 必读文件清单
   - 列出你要读取的 base、plugin、patch、学习卡片、知识卡片、旧题材料和模板；
   - 标明哪些文件已经存在，哪些缺失。

3. 自动选题计划
   - 扫描哪些目录；
   - 如何识别官方题面、附件、结果模板、优秀论文、解析和学习卡片；
   - 如何判断 T0/T1/T2/T3/T4；
   - 如何判断 M1/M2/M3/M4/M5；
   - 如何生成候选题排序表；
   - 如何选择本轮题目。

4. 官方材料自动获取计划
   - 本地没有 T3 时，去哪里找官方题面、附件和模板；
   - 如何判断来源是否官方或可信镜像；
   - 下载后保存到哪里；
   - 如何生成 `manifest.md`；
   - 如何避免读取优秀论文、答案、解析和结果数值；
   - 如果下载失败，如何生成官方材料获取失败报告。

5. 本轮材料资格审查计划
   - 是否来自官方题面；
   - 是否有官方附件；
   - 是否有输出模板；
   - 是否与 patch 来源论文同题；
   - 是否有答案泄漏风险；
   - 是否允许计入 stable。

6. 总控诊断执行计划
   - 只做总控诊断；
   - 不写论文；
   - 不写代码；
   - 不给最终答案；
   - 必须输出训练材料理解、材料资格审查、旧题总控诊断、优化专项检查、模型候选与质疑、数据需求、图表计划、人工确认项、100 分评分、失败标签和闸门判定。

7. 评分和闸门计划
   - 如何按 100 分评分；
   - 如何检查 P1-P10；
   - 如何检查 M1-M5；
   - 什么情况下生成失败复盘；
   - 什么情况下生成提示词修复草案；
   - 什么情况下生成重测任务；
   - 什么情况下允许计入 stable。

8. 输出文件计划
   - 候选题排序表；
   - 闭环摘要；
   - 旧题测试记录；
   - 失败复盘卡，如需要；
   - 提示词修复草案，如需要；
   - 重测任务，如需要；
   - training_log.md 更新计划。

9. 防遗忘清单
   - T0/T1/T2 不计入 stable；
   - 同题优秀论文样本不计入 stable；
   - M1/M2/M3/M5 不计入 stable；
   - 没有官方题面和附件不得伪装成 T3；
   - 本地没有 T3 时必须先尝试自动获取官方材料；
   - 自动获取失败才允许要求人工补材料；
   - 不能因为总分 >= 80 就进入求解；
   - 不能未经人工确认修改正式 base/plugin/patch；
   - 不能重复追加 training_log.md 已有记录。

输出计划后，等待自己检查计划是否完整。
计划完整后，再按计划执行。
```

## 6. 执行时的短上下文

如果上下文太长，AI 应优先保留以下短上下文：

```text
当前任务：自动完成数模旧题总控诊断闭环测试。

核心禁止：
不写论文，不写代码，不给最终答案，不跳过总控诊断，不乱改正式提示词。

材料等级：
T0 流程冒烟，不计 stable。
T1 材料受限，不计 stable。
T2 同题官方题面，不计 stable。
T3 独立官方泛化测试，可计 stable。
T4 稳定候选测试，可建议 stable。

风险标签：
M1 答案泄漏。
M2 附件/模板缺失。
M3 同题污染。
M4 经验照搬。
M5 人工补题意过多。

stable 条件：
至少 2 道不同题号 T3/T4 通过，且无 P3/P4/P5/M1/M2/M3/M5。

输出文件：
候选题排序表、闭环摘要、旧题测试记录、必要时失败复盘、必要时修复草案、必要时重测任务、training_log 更新。
```

## 7. 什么时候停止

AI 只有在以下情况可以停止并要求人工补充：

1. 本地没有任何可用题面；
2. 本地没有 T3，且已经尝试自动获取官方题面、附件和模板但失败；
3. 无法判断文件是否官方材料，且无法通过来源校验确认；
4. 网络、权限或下载失败，且无法使用国内镜像或可信学校镜像；
5. 需要正式修改 base/plugin/patch；
6. 需要正式标记 stable。

停止时必须输出：

```text
停止原因：
已完成动作：
缺失材料：
需要人工提供什么：
提供后从哪一步继续：
```

## 8. 完成标准

一次计划驱动闭环必须完成：

1. 先生成本轮执行计划；
2. 再执行候选题扫描或读取指定旧题；
3. 若没有 T3，先自动尝试获取官方材料；
4. 再完成材料资格审查；
5. 再运行总控诊断；
6. 再评分和闸门判定；
7. 再写入对应文件；
8. 最后输出闭环报告。

如果没有计划，闭环无效。
