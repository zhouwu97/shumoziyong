# 数学建模项目 v2.1 全链路闭环

本流程在保留 Runtime 1.1/1.2 与历史运行解释的前提下，为新运行启用 Runtime 1.3.0 和 Gate Contract 2.1.0。旧运行不迁移、不覆盖、不重算历史分数。

## 固定执行顺序

```text
Gate 0 → Gate 1 合同 → Gate 2 执行规范
→ Python 求解与 active Formal Result
→ MATLAB Level A+B 独立复算
→ Gate 3 三维判定与 Paper Admission
→ Gate 4 论文候选稿、Python 图表、Reviewer A/B Round 1
→ 局部修复并重跑受影响证据
→ Reviewer A/B Round 2
→ Gate 5 最终验收、评分与封存
```

未通过 Paper Admission 时，`paper_production_manifest.paper_type` 必须为 `technical_report`，不得生成 `submission_paper`。

## 2024-C 证据身份

2024-C 固定标记为：

```json
{
  "classification": "development_integration_benchmark",
  "blind_generalization": false,
  "profile_promotion_eligible": false
}
```

它只用于验证合同、闸门和论文生产链能否完整运行，不构成陌生题泛化证据。

## MATLAB 复算命令

在当前 Run 中准备输入合同后执行：

```powershell
python scripts/run_workflow.py matlab-recompute `
  --run-dir runs/<run_id> `
  --level A `
  --input runs/<run_id>/matlab_level_a_input.json

python scripts/run_workflow.py matlab-recompute `
  --run-dir runs/<run_id> `
  --level B `
  --input runs/<run_id>/matlab_level_b_input.json
```

MATLAB 不可用、输入缺失、哈希不一致或数值检查失败时，只生成 blocker，不得把 Python 或 Octave 结果标成 MATLAB 证据。

## 论文与图表

论文先建立一句话核心论点、Terminology Ledger、Claim-Evidence Map 和结论边界，再生成候选稿。图表必须由 Python 生成，且同时保存 source data、脚本、SVG、PDF、TIFF 和视觉 QA；MATLAB 只负责独立复算，不参与正式绘图。

```powershell
python scripts/paper/render_v21_figure.py `
  --run-dir runs/<run_id> `
  --contract runs/<run_id>/paper/figure_contracts/figure_001.json

python scripts/paper/build_v21_production_manifest.py `
  --run-dir runs/<run_id> `
  --one-sentence-argument "..." `
  --terminology-ledger paper/terminology_ledger.md `
  --claim-map paper_claim_map.json `
  --manuscript paper/manuscript.md `
  --pdf paper/manuscript.pdf `
  --figure-fragment figures/figure_001.fragment.json
```

所有引用均绑定当前 Run 内文件的 SHA-256。缺失证据只能进入 `missing_evidence_placeholders`，不得补写数值或结论。
