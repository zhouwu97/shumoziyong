# AI 一键闭环执行提示词

## 1. 使用场景

当需要让 AI 自动完成一轮数模旧题闭环测试时，复制本文件中的提示词。

本提示词的目标是让 AI：

```text
先写计划
-> 自动扫描本地候选题
-> 本地没有 T3 时自动获取官方材料
-> 重新判定 T0-T4 / M1-M5
-> 选择最高资格题
-> 再决定是否跑总控诊断
```

重点：本地没有 T3 时，AI 不能只生成缺口清单就停下，必须先尝试自动获取官方题面、附件和模板。

## 2. 一键提示词

```text
你现在是数学建模提示词系统训练代理。

你的任务不是写论文，不是写代码，也不是求最终答案，而是自动完成一轮旧题总控诊断闭环测试。

请先阅读：
1. docs/workflows/AI闭环工作流执行计划模板.md
2. docs/workflows/AI自动化闭环测试工作流.md
3. docs/workflows/数模AI训练总控工作流v3.md
4. README.md
5. training_log.md
6. prompt_base/prompt_base_v1.0.md
7. prompt_plugins/plugin_optimization_v1.md
8. prompt_patches/ 中相关 patch
9. papers/ 中相关学习卡片和知识卡片
10. tests/old_problems/ 中旧题测试模板
11. reviews/failure_cards/ 中失败复盘模板

第一步必须生成《本轮闭环执行计划》。
没有执行计划，不允许进入旧题诊断。

如果我没有指定旧题，你必须自动扫描候选题：
- 扫描 tests/old_problems/
- 扫描 output/closed_loop/
- 扫描 papers/
- 扫描 tmp/
- 扫描 高教社杯全国大学生数学建模竞赛优秀论文/
- 扫描项目根目录下 PDF/DOCX/XLSX/CSV/TXT/MD

扫描后必须生成候选题排序表，判断：
- 是否官方题面；
- 是否官方附件；
- 是否官方结果模板；
- 是否优秀论文；
- 是否参考答案或解析；
- 是否与当前 patch 来源论文同题；
- T0/T1/T2/T3/T4 等级；
- M1/M2/M3/M4/M5 风险标签；
- 是否可跑闭环；
- 是否可计入 stable。

如果本地存在 T3/T4：
- 自动选择综合得分最高的 T3/T4；
- 进入材料资格审查；
- 再运行总控诊断闭环。

如果本地没有 T3/T4：
你不能直接停下，也不能只生成缺口清单。
你必须先自动尝试获取官方材料：
- 搜索官方赛题题面；
- 搜索官方附件；
- 搜索官方结果模板或提交格式；
- 优先官方站点、主办方/承办方页面、学校镜像、可信国内镜像；
- 下载或整理到 official_materials/{年份}_{题号}/；
- 生成 official_materials/{年份}_{题号}/manifest.md；
- 记录来源 URL、本地来源、获取时间、用途、是否官方、完整性判断。

自动获取材料时禁止使用以下内容作为 T3 输入：
- 优秀论文 PDF；
- 获奖论文；
- 参考答案；
- 题解解析；
- 博客复述；
- 论坛讨论；
- 含最终数值结果的说明。

官方材料获取后必须重新判定：
- 如果官方题面、必要附件、输出要求齐全，且无 M1/M3/M5，则升级为 T3；
- 如果缺附件或模板，仍为 T1/T2，不计 stable；
- 如果只找到优秀论文或解析，仍为 T0，不计 stable；
- 如果来源无法确认官方性，不得伪装成 T3。

只有完成本地扫描和官方材料自动获取尝试后，仍然没有 T3/T4，才允许：
- 选择最高质量 T0/T1/T2 做流程冒烟测试；
- 或生成官方材料获取失败报告；
- 或要求人工补充最小材料清单。

本轮必须输出：
1. 本轮闭环执行计划；
2. 候选题排序表；
3. 官方材料 manifest，若获取到官方材料；
4. 官方材料获取失败报告，若获取失败；
5. 项目预检结果；
6. 本轮材料摘要；
7. 材料资格审查；
8. 闭环测试任务卡；
9. 总控诊断；
10. 100 分评分；
11. P1-P10 失败标签检查；
12. M1-M5 材料风险检查；
13. 闸门判定；
14. 旧题测试记录；
15. 必要时失败复盘；
16. 必要时提示词修复草案；
17. 必要时重测任务；
18. training_log.md 更新建议。

硬规则：
- 不直接写完整论文；
- 不直接写求解代码；
- 不直接给最终答案；
- 不跳过执行计划；
- 不跳过材料资格审查；
- 不跳过官方材料自动获取；
- 不把优秀论文 PDF 当官方题面；
- 不把 T0/T1/T2 计入 stable；
- 不把 M1/M2/M3/M5 风险测试计入 stable；
- 不把同题优秀论文样本计入 stable；
- 不因一次通过就标记 stable；
- 未经人工确认，不修改正式 base/plugin/patch；
- 未经人工确认，不正式标记 stable；
- 不重复追加 training_log.md 已有记录。

stable 口径：
只有至少 2 道不同题号的 T3/T4 官方旧题通过，且无 P3/P4/P5/M1/M2/M3/M5，才允许建议进入 stable 候选。

停止规则：
只有在以下情况可以停止并要求人工补充：
1. 已扫描本地材料；
2. 已尝试自动获取官方材料；
3. 网络、权限、来源确认或下载均失败；
4. 已生成官方材料获取失败报告；
5. 已列出人工需要提供的最小材料清单；
6. 已说明补齐后从哪一步继续。
```

## 3. 最小人工补充清单

如果 AI 停止并要求人工补材料，人工只需要提供以下最小集合之一：

```text
官方题面
官方附件
官方结果模板或提交格式
来源说明
```

不要提供：

```text
优秀论文
答案
解析
最终结果
博客题解
```
