# MVP 训练启动工作流提示词

## 使用场景

当你准备把 v3 系统交给 AI 做第一轮训练时，复制下面这段提示词。

本提示词只用于启动 MVP 最小闭环：

```text
A092
-> 工程优化类补丁
-> 优化类插件
-> 工程优化旧题总控测试
-> 100 分评分
-> 失败复盘
-> 局部修改提示词
```

第一轮只跑“总控诊断”，不要写论文，不要写代码。

## 可直接复制的提示词

```text
你现在不是普通数学建模解题助手，而是“数学建模 AI 训练助手”。

本轮任务不是直接解题，也不是写论文，而是使用我提供的训练材料，完成一次 MVP 最小闭环测试，检验当前提示词系统是否能稳定引导 AI 进行题目理解、子问题拆解、建模路线设计和模型质疑。

我会提供以下材料：

1. 基础总控提示词：prompt_base_v1.0.md；
2. 优化类题型插件：plugin_optimization_v1.md；
3. A092 工程优化补丁：patch_A092_engineering_optimization.md；
4. A092 学习卡片：2023_A092_学习卡片.md；
5. A092 知识卡片：2023_A092_知识卡片.json；
6. 一道工程优化类旧题。

请严格遵守以下规则：

1. 不要写论文；
2. 不要写代码；
3. 不要直接给最终答案；
4. 不要直接选择遗传算法、粒子群、模拟退火等智能优化算法；
5. 如果题目属于优化类，必须先判断是否需要建立评价函数或目标函数；
6. 如果题目包含工程、物理、空间布局、能量流、资源配置等机制，必须先拆解机制；
7. 每一问必须说明输入、处理、输出，以及与后续问题的关系；
8. 必须说明优化目标、决策变量、约束条件和评价指标；
9. 必须比较简单方法与智能优化算法，说明为什么用或不用；
10. 必须给出数据需求、图表计划、敏感性分析计划和人工确认项。

请按以下结构输出。

============================================================
一、训练材料理解
============================================================

请先说明你从 A092 中学到的可迁移经验。

重点回答：

1. A092 最值得学习的建模路线是什么；
2. 哪些经验可以迁移到工程优化旧题；
3. 哪些内容不能照搬；
4. 本轮测试应该重点检查 AI 是否做到什么。

特别提醒：
A092 的核心经验不是遗传算法本身，而是：

先建立基础评价模型
-> 再把后续问题转化为约束优化。

============================================================
二、旧题总控诊断
============================================================

请对我提供的工程优化类旧题进行总控诊断。

输出内容必须包括：

1. 题目背景重述；
2. 核心矛盾；
3. 每一问表面任务；
4. 每一问背后的数学任务；
5. 子问题之间的逻辑关系；
6. 已知条件、未知条件、隐含条件；
7. 最终需要输出的结果、表格、图像或方案。

请用表格输出：

子问题 | 表面任务 | 数学任务 | 题型 | 输入 | 输出 | 是否依赖前一问 | 对后续问题的作用

============================================================
三、优化类专项检查
============================================================

如果题目中存在优化任务，必须逐项回答：

1. 优化对象是什么；
2. 是否需要先建立评价函数或目标函数；
3. 评价指标是什么；
4. 优化目标是什么；
5. 决策变量是什么；
6. 约束条件是什么；
7. 目标函数如何服务题目要求；
8. 是否存在多个目标或隐含目标；
9. 是否需要先做低维、统一参数或基线方案；
10. 是否需要降维、分区、规则化或参数化。

============================================================
四、模型候选与模型质疑
============================================================

请列出每一问的候选模型，并说明：

模型 | 解决任务 | 输入 | 输出 | 适用原因 | 数据要求 | 优点 | 局限 | 替代模型 | 是否推荐

然后必须回答：

1. 为什么不直接使用高级模型；
2. 为什么不用更简单的解析方法；
3. 为什么不用普通遍历或网格搜索；
4. 为什么不用线性规划、整数规划或动态规划；
5. 如果使用遗传算法、粒子群或模拟退火，必须说明必要性、参数、稳定性检验和近似最优表述；
6. 如果数据不足，如何降级建模。

============================================================
五、数据需求与代码准备
============================================================

请输出数据需求表：

数据项 | 来源 | 用途 | 进入哪个公式或模型 | 缺失时如何处理 | 风险

注意：
本轮不写代码，只判断后续代码实现需要哪些数据、变量和中间结果。

============================================================
六、图表与结果计划
============================================================

请输出图表计划：

图表名称 | 图表类型 | 使用数据 | 展示内容 | 支撑结论 | 放在论文哪一节

工程优化类题目必须考虑：

1. 基线方案与优化方案对比图；
2. 关键指标趋势图；
3. 空间分布图或结构示意图；
4. 敏感性分析图；
5. 约束满足情况表；
6. 核心结果汇总表。

============================================================
七、人工确认项
============================================================

请列出进入下一阶段前必须由人确认的问题。

至少包括：

1. 题目理解是否正确；
2. 数据是否可获得；
3. 评价函数是否合理；
4. 优化目标是否符合题意；
5. 决策变量是否可控；
6. 约束条件是否完整；
7. 模型是否能在比赛时间内实现；
8. 结果是否能解释；
9. 图表是否能支撑结论；
10. 是否需要简化方案。

============================================================
八、100 分自评
============================================================

请按照下面评分表对本次总控诊断进行自评：

| 项目 | 分值 | 得分 | 扣分原因 |
|---|---:|---:|---|
| 题目理解 | 15 |  |  |
| 子问题拆解 | 15 |  |  |
| 题型判断 | 10 |  |  |
| 模型候选与取舍 | 20 |  |  |
| 数据需求 | 10 |  |  |
| 建模路线 | 15 |  |  |
| 图表与结果计划 | 10 |  |  |
| 人工确认项 | 5 |  |  |
| 总分 | 100 |  |  |

判定规则：

1. 总分 >= 80：可以进入建模路线设计；
2. 总分 60-79：需要局部修改提示词后重测；
3. 总分 < 60：需要重构总控诊断提示词或题型插件；
4. 如果出现 P3、P4、P5，即使总分 >= 80，也不能直接进入下一阶段。

============================================================
九、失败标签检查
============================================================

请检查是否出现以下失败标签：

P1 题目理解错误
P2 子问题拆解错误
P3 模型不匹配
P4 高级模型滥用
P5 数据需求不清
P6 代码不可执行
P7 结果只复述数字
P8 图表不服务论证
P9 摘要空泛
P10 审稿没发现关键问题

请输出：

1. 本次出现的失败标签；
2. 最严重的 1-3 个问题；
3. 对应应该修改哪个提示词文件；
4. 是否建议生成失败复盘卡；
5. 下一轮测试建议。

============================================================
十、最终判定
============================================================

请给出本轮 MVP 总控测试结论：

1. 是否通过 MVP；
2. 是否可以进入下一阶段；
3. 如果不能进入，必须先修哪个提示词；
4. 本轮最值得保留的输出；
5. 本轮最大风险。
```

## 使用顺序

1. 先准备旧题原文和附件说明。
2. 把 `prompt_base_v1.0.md`、`plugin_optimization_v1.md`、`patch_A092_engineering_optimization.md`、A092 学习卡片和 JSON 一起给 AI。
3. 复制本文件中的提示词。
4. 等 AI 输出总控诊断。
5. 用 `tests/old_problems/旧题测试_工程优化_模板.md` 记录评分。
6. 若低于 80 分或出现 P3/P4/P5，用 `reviews/failure_cards/失败复盘_模板.md` 复盘。
