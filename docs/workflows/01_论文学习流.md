# 论文学习流

## 使用场景

只在学习优秀论文时使用。目标是提炼可迁移经验，生成 patch 草案，而不是直接修改正式提示词。

## 入口提示词

```text
你现在执行“论文学习流”。

请读取：
1. docs/workflows/01_论文学习流.md
2. prompt_base/prompt_base_v1.0.md
3. papers/templates/学习卡片模板.md
4. papers/templates/知识卡片模板.json
5. prompt_patches/templates/patch模板.md
6. 当前优秀论文材料

任务：
只提炼可迁移经验，生成学习卡片、知识卡片 JSON 和 patch 草案，并给出 patch_index 登记建议。

禁止：
1. 不修改 base；
2. 不修改 plugin；
3. 不标记 `competition_evidenced`；
4. 不把单篇论文经验当通用规则；
5. 不跑旧题闭环；
6. 不进入新题解题。
```

## 执行步骤

1. 核对论文来源、年份、题号和题型。
2. 提取题目理解方式、子问题拆解、建模路线、模型选择理由。
3. 提取数据处理、图表论证、结果解释和模型评价方式。
4. 区分可迁移经验与不可照搬内容。
5. 生成 `papers/{年份}_{编号}_学习卡片.md`。
6. 生成 `papers/{年份}_{编号}_知识卡片.json`。
7. 生成 `prompt_patches/patch_{编号}_{主题}.md` 草案，或给出 patch 草案内容。
8. 给出 `prompt_patches/patch_index.json` 登记建议，默认状态为 `review_ready`。
9. 给出 `training_log.md` 更新建议。

## 输出要求

```text
1. 学习卡片
2. 知识卡片 JSON
3. patch 草案
4. 不可照搬清单
5. patch_index 登记建议
6. training_log 更新建议
```

## 质量检查

- 经验是否脱离了论文具体数值、公式、代码和结论。
- patch 是否只描述单篇论文经验。
- 是否避免把单篇论文经验提升到 base。
- 是否说明适用条件和误用风险。
- 是否明确 `review_ready` patch 需要完整回归证据后才可升级。
