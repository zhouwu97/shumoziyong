# 论文学习流

## 使用场景

用于学习优秀论文，也用于记录“外观优秀但证据薄弱”的反例论文和与既有规则冲突的论文。目标是提炼可迁移经验、失败边界与反证，形成可追溯资产，而不是把单篇论文直接写成全局真理。

## 入口提示词

```text
你现在执行“论文学习流”。

请读取：
1. docs/workflows/01_论文学习流.md
2. prompt_base/prompt_base_v1.0.md
3. papers/templates/学习卡片模板.md
4. papers/templates/知识卡片模板.json
5. prompt_patches/templates/patch模板.md
6. papers/EXCELLENT_PAPER_REVIEW_STANDARD_REGISTRY.json
7. docs/contest_v2/NATIONAL_CONTEST_REVIEW_WORKFLOW.md
8. 当前优秀论文材料

任务：
只提炼可迁移经验，生成学习卡片、知识卡片 JSON 和 patch 草案；把论文来源及候选评审规则写入优秀论文评审标准注册表，并更新训练日志。

禁止：
1. 不修改 base；
2. 不修改 plugin；
3. 不标记 `competition_evidenced`；
4. 不把单篇论文经验当通用规则；
5. 不跑旧题闭环；
6. 不进入新题解题。
7. 未核验 claim 不得升级为全局硬规则。
8. 不得用同题答案、具体数值或结论反向约束新题。
9. 语义重复规则不得新建 patch；只补充证据、适用范围、反例或冲突。
10. diagnostic 旧题运行不得用于 `validated_candidate` 晋级。
```

## 执行步骤

1. 核对论文来源、年份、题号、题型、文件摘要和可读取页码；PDF 必须检查完整性、目录、图表、公式与版式，不能只读抽取文本。
2. 为每条重要经验记录页码、证据摘要、核验状态和证据强度；无法定位到原文的内容标为 `unverified`。
3. 提取题目理解方式、子问题拆解、建模路线、模型选择理由、基线、验证、结果解释和局限性；若为反例，明确“看似优秀的表面特征”和“实际失效证据”。
4. 提取图表如何支撑论证，不把“图多”误写成质量规则。
5. 区分可迁移经验、适用范围、误用风险、不可迁移内容和同题泄漏风险。
6. 生成 `papers/{年份}_{编号}_学习卡片.md`。
7. 生成 `papers/{年份}_{编号}_知识卡片.json`，必须填写 `transfer_scope`、`misuse_risk`、`non_transferable` 与 `review_standard_candidates`。
8. 在生成 patch 前，对 `patch_index.json`、现有 patch 和注册表规则做语义去重。若已有同义规则，不新建 patch，只把新来源、适用范围、反例或冲突追加到现有规则的证据记录。
9. 仅在确有新机制时生成 `prompt_patches/patch_{编号}_{主题}.md` 草案并更新 `patch_index.json`，默认状态为 `review_ready`。
10. 更新 `papers/EXCELLENT_PAPER_REVIEW_STANDARD_REGISTRY.json`：登记来源、核验状态、允许用途和候选规则。
11. 更新 `training_log.md`，记录本次学习产物、核验状态和下一步验证；不是只给建议。
12. 对照已有论文做跨论文冲突检查，主动登记以下反例：复杂模型不如廉价基线、图表很多但没有论证、奖项高但方法不可迁移、方法外观漂亮但验证无效、结论强度超过求解/样本证据。
13. `validated_candidate` 证据只能来自 `docs/workflows/02_旧题闭环流.md` 的 `production_replay` 或陌生新题的完整生产执行；diagnostic、Prompt 回归、文件生成或旧 Gate 通过均不合格。
14. 只有已核验、通过生产回归且获得跨论文或跨题支持的机制，才可提议升级为 `global_active`。
15. 若规则获准升级，同步更新 `docs/contest_v2/NATIONAL_CONTEST_REVIEW_WORKFLOW.md` 的版本和变更记录；不得静默覆盖旧标准。

## 规则晋级状态

```text
single_paper_candidate
→ cross_paper_candidate
→ validated_candidate
→ global_active
→ retired
```

- `single_paper_candidate`：只在该论文学习卡和定向评阅中使用。
- `cross_paper_candidate`：多个独立论文出现，但尚未完成旧题验证。
- `validated_candidate`：有无泄漏的 `production_replay` 或陌生题完整执行证据，且改善体现在模型、求解、论证或论文质量；等待纳入全局标准。
- `global_active`：可进入新题作者自检和独立 Reviewer 的通用标准。
- `retired`：被反例、冲突证据或更准确规则替代，保留来源和退役原因。

自动写入只包括学习资产和候选规则。任何单篇、未核验或与题型强绑定的经验，都不得自动晋级为 `global_active`。

## 输出要求

```text
1. 学习卡片
2. 知识卡片 JSON
3. patch 草案
4. 不可照搬清单
5. patch_index 实际登记
6. 优秀论文评审标准注册表实际登记
7. training_log 实际更新
8. 候选规则的晋级/保留理由
9. 语义去重结果：新建、合并或仅补证据
10. 反例、冲突与规则失效条件
```

## 质量检查

- 经验是否脱离了论文具体数值、公式、代码和结论。
- patch 是否只描述单篇论文经验。
- 是否避免把单篇论文经验提升到 base。
- 是否说明适用条件和误用风险。
- 是否明确 `review_ready` patch 需要完整回归证据后才可升级。
- 是否为关键经验提供了页码或可复核证据，而不是仅凭摘要印象。
- 是否记录来源核验状态、适用范围、误用风险和不可迁移项。
- 是否完成注册表与训练日志写入，并保持 JSON 可解析。
- 是否避免把语言风格、算法名称或奖项标签误当成可迁移创新。
- 是否先做语义去重，避免同义 patch 膨胀。
- 是否记录反例和冲突证据，而不是只累计正面模式。
- 晋级证据是否来自 production replay 或陌生题完整生产，而非 diagnostic。
