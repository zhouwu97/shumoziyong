# 失败标签 P1-P10

P 标签用于判断总控诊断是否失败，以及失败应修哪个层级。

| 标签 | 含义 | 严重程度 |
|---|---|---|
| P1 | 题目理解错误 | 高 |
| P2 | 子问题拆解错误 | 高 |
| P3 | 模型不匹配 | 最高 |
| P4 | 高级模型滥用 | 最高 |
| P5 | 数据需求不清 | 最高 |
| P6 | 代码不可执行 | 中，本阶段通常不触发 |
| P7 | 结果只复述数字 | 中 |
| P8 | 图表不服务论证 | 中 |
| P9 | 摘要空泛 | 低，本阶段通常不触发 |
| P10 | 审稿没发现关键问题 | 中 |

## 闸门规则

- 出现 P3/P4/P5 时，本轮总控诊断不通过。
- 总分低于 80 时，本轮总控诊断不通过。
- 出现 P6/P7/P8/P9/P10 时，按阶段判断是否生成复盘或修复草案。
- 如果问题只来自材料缺失，优先补材料，不急着改提示词。

## 修复层级

| 问题来源 | 优先修复层级 |
|---|---|
| 所有题型都应遵守的总控闸门缺失 | base |
| 某一题型的目标、变量、约束或算法规则缺失 | plugin |
| 单篇论文经验迁移不到位或被照搬 | patch |
| 本次输出执行不到位 | 不改提示词，重测 |
| 旧题材料不完整 | 补材料，重测 |
