# 旧题闭环流

## 使用场景

用于赛前测试提示词稳定性。这个流程只做旧题总控诊断闭环，不写代码、不写论文、不求最终答案。

## 必读文件

```text
1. docs/workflows/02_旧题闭环流.md
2. docs/workflows/rules/材料等级_T0-T4.md
3. docs/workflows/rules/材料风险_M1-M5.md
4. docs/workflows/rules/失败标签_P1-P10.md
5. docs/workflows/rules/总控诊断评分表.md
6. docs/workflows/rules/stable判定规则.md
7. training_log.md
8. prompt_base/prompt_base_v1.0.md
9. 对应 prompt_plugins/
10. 对应 prompt_patches/
11. tests/old_problems/ 或 official_materials/
```

## 入口提示词

```text
你现在执行“旧题闭环流”。

任务：
完成一轮旧题总控诊断闭环测试。

第一步必须生成《本轮闭环执行计划》。

如果我没有指定旧题，请自动扫描候选题。
优先使用 T3/T4 官方材料。
本轮不写代码、不写论文、不直接给最终答案。

请读取 docs/workflows/rules/ 下的规则文件，不要在入口中重复规则全文。
```

## 执行步骤

1. 生成《本轮闭环执行计划》。
2. 扫描候选旧题材料。
3. 判断材料等级 T0-T4。
4. 判断材料风险 M1-M5。
5. 选择当前最高资格旧题。
6. 如果本地没有 T3/T4，先尝试获取官方题面、附件和输出模板。
7. 读取 base、plugin、patch 和训练日志。
8. 生成材料资格审查。
9. 生成旧题总控诊断。
10. 按 100 分评分。
11. 检查 P1-P10。
12. 输出闸门判定。
13. 生成旧题测试记录。
14. 如失败，生成失败复盘。
15. 如需修改提示词，只生成修复草案。
16. 给出 `training_log.md` 更新建议。

## 必须输出

```text
1. 本轮执行计划
2. 候选题排序表
3. 材料资格审查
4. 总控诊断
5. 100 分评分
6. P1-P10 检查
7. M1-M5 检查
8. 闸门判定
9. 旧题测试记录
10. 失败复盘或修复草案
11. training_log 更新建议
```

## 禁止事项

- 不直接写完整论文。
- 不直接写求解代码。
- 不直接给最终答案。
- 不跳过执行计划。
- 不跳过材料资格审查。
- 不把 T0/T1/T2 计入 stable。
- 不未经人工确认修改正式 base/plugin/patch。
- 不未经人工确认正式标记 stable。
