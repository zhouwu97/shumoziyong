# 旧题闭环流

## 1. 定位

本入口包含两个互斥模式：

```text
diagnostic          旧题诊断，不生产答案
production_replay   旧题真实生产回归，完整建模与论文
```

必须在开始时声明模式。两种模式的产物和证据等级不能混用。

## 2. diagnostic：旧题诊断模式

### 用途

用于检查材料资格、Prompt 选择、题意拆解、风险标签和工作流行为。它不是“完整复现”，不验证实际求解、论文写作或提交能力。

### 入口提示词

```text
请执行 docs/workflows/02_旧题闭环流.md，模式 diagnostic。

扫描或读取指定旧题，完成材料资格、总控诊断、Prompt/patch 选择、风险和失败标签检查。
不写求解代码、不生成最终答案、不写完整论文，也不得把本次结果作为 validated_candidate、
建模能力、论文能力或竞赛资格证据。
```

### 执行步骤

1. 生成本轮诊断计划。
2. 扫描候选题并判断材料等级 T0–T4、材料风险 M1–M5。
3. 优先选择官方 T3/T4 材料；材料不足时明确降级。
4. 读取 base、plugin、patch 索引和训练日志。
5. 记录启用的 patch 及其状态，检查同题泄漏。
6. 完成题意、变量、约束、数据、验证方案和潜在失败的总控诊断。
7. 按现有诊断表评分并检查 P1–P10。
8. 写测试记录、失败复盘或修复草案。
9. 更新训练日志，但证据类型必须标为 `diagnostic_only`。

### 禁止事项

- 不写求解代码、最终数值答案或完整论文；
- 不进入 Paper Admission 或 Reviewer 评分；
- 不把 Prompt 行为正确解释为建模正确；
- 不将本模式用于任何学习规则的 `validated_candidate` 晋级；
- 不把 T0/T1/T2、轻量回归、旧 Gate 通过或编排成功作为生产证据。

## 3. production_replay：真实生产回归模式

### 用途

用于验证新学习规则、跨题泛化和完整生产能力。它必须像陌生新题一样从官方材料完成模型、代码、结果、图表、论文、工程验收、Paper Admission 和独立评审。

### 入口提示词

```text
请执行 docs/workflows/02_旧题闭环流.md，模式 production_replay。

使用指定官方旧题，但不得读取同题优秀论文、参考答案、历史 Formal Result、旧正文、旧结果或
题目专用求解器。调用 docs/workflows/03_新题执行流.md 的 standard 生产主链，完成全部问题的
模型、代码、Result、Verification、Ledger、图表、完整论文、Paper Admission 和独立 Reviewer。

记录本次使用的学习规则、对照版本、暴露的问题和是否真正提高模型/论文质量。
```

### 输入冻结

开始前必须冻结：

- 题目编号、官方题面、附件和模板；
- 允许使用的通用工具、base、plugin、patch 与优秀论文评审规则版本；
- 禁止读取的同题答案、旧结果、旧论文和专用代码；
- 计算时间、内存、求解停止条件和总执行预算；
- 评价指标、廉价基线和拟验证的学习规则。

不得运行后改选更容易的题目或补读同题材料。

### 执行步骤

1. 完成材料资格与泄漏审计；只允许官方材料和冻结的通用能力。
2. 调用 `03_新题执行流.md` 的 `standard` 模式，不复制一套生产流程。
3. 默认建立一个主模型和一个廉价基线；按题型完成有效验证。
4. 生成每问 Result/Verification、Ledger、`results.typ`、图表、正文、PDF 和 package。
5. 分别记录 `ENGINEERING_VERIFICATION` 与 `PAPER_ADMISSION`，不得用前者替代后者。
6. Paper Admission PASS 后由 AI 编排层自动新建独立 Reviewer 对话；修补 MUST 后自动新建全新复审对话。
7. 比较启用/不启用候选学习规则时的模型质量、求解证据、论文论证或失败模式；不能只比较文件是否生成。
8. 更新训练日志并登记运行目录、Reviewer 结论和剩余风险。

### 可作为规则验证证据的最低条件

只有同时满足以下条件，本次运行才能支持 `validated_candidate`：

- 官方 T3/T4 材料完整且输入冻结；
- 无同题答案、旧结果、旧正文或专用代码泄漏；
- 全部必答问题形成真实模型、代码、结果、图表和完整论文；
- `ENGINEERING_VERIFICATION=PASS`；
- `PAPER_ADMISSION=PASS`；
- 独立 Reviewer 对最终文件至少给出 `SUBMISSION_RECOMMENDED`，且无 MUST；
- 候选规则带来可定位的改善，或成功避免了预先声明的失败；
- 改善不是由增加题目专用特判、放宽检查或读取历史答案造成。

单次 production replay 只能提供候选证据，不能自动证明稳定泛化或奖项水平。

## 4. 与旧 Formal 系统的关系

历史 Gate、Runtime、Manifest、Seal 与 capability evidence 只在用户明确指定 `formal_audit` 时用于历史复现。普通 diagnostic 和 production_replay 不调用旧 `04_v2.1` 流程，也不写旧资格状态。

## 5. 可信边界

- diagnostic 只验证诊断与提示词行为；
- production_replay 才验证实际建模、计算和论文生产；
- 工程 PASS 不等于论文准入；
- Paper Admission 不等于独立 Reviewer 推荐；
- 单道旧题通过不等于跨题稳定能力。
