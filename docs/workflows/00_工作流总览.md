# 工作流总览

本目录只保留三条主流程入口：

```text
01_论文学习流.md
02_旧题闭环流.md
03_新题执行流.md
```

三条流程不能混用。入口文件只负责调度，不复制 base、plugin、patch 或 rules 的全文。

## 1. 论文学习流

使用场景：日常学习优秀论文。

目标：

```text
优秀论文 PDF
-> 学习卡片
-> 知识卡片 JSON
-> patch 草案
-> patch_index 登记建议
-> training_log 更新建议
```

边界：

- 不修改正式 `prompt_base/`。
- 不修改正式 `prompt_plugins/`。
- 不标记 `competition_evidenced`。
- 不把单篇论文经验当通用规则。

## 2. 旧题闭环流

使用场景：赛前验证提示词稳定性。

目标：

```text
执行计划
-> 候选旧题扫描
-> 材料资格审查
-> 总控诊断
-> 评分与失败标签
-> 测试记录 / 复盘 / 修复草案
-> training_log 更新建议
```

边界：

- 不写完整论文。
- 不写求解代码。
- 不给最终答案。
- 不未经人工确认修改正式 base/plugin/patch。
- 不把轻量 Prompt 回归或未封存运行解释为晋级证据。

## 3. 新题执行流

使用场景：比赛当天或模拟赛当天。

目标：

```text
新题
-> 总控诊断
-> 人工确认路线
-> 建模设计
-> 代码与结果
-> 论文写作
-> 审稿反查
```

边界：

- 第一轮只做总控诊断。
- 路线确认前不写代码。
- 结果确认前不写论文。
- 新题模式只改变人工确认点，不取消 Gate 0-5 产物和机器校验。

## 4. 文件职责

| 位置 | 职责 |
|---|---|
| `README.md` | 项目定位、目录说明、三个入口 |
| `docs/workflows/` | 三条流程的调度入口 |
| `docs/workflows/rules/` | 旧题闭环的判定规则 |
| `prompt_base/` | 通用总控诊断规则 |
| `prompt_plugins/` | 题型专项规则 |
| `prompt_patches/` | 单篇论文经验补丁 |
| `prompt_patches/patch_index.json` | patch 检索、筛选和 runtime 导出索引 |
| `papers/templates/` | 论文学习卡片和知识卡片模板 |
| `training_log.md` | 训练和测试事实记录 |
