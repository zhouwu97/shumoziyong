# CUMCM 2026 论文格式与内容清单

这是作者侧交稿前的实用清单，不是组委会官方评分细则。当前 `contest_v2` 初始化模板独立实现，参考 `jihe520/MathModelAgent` 的章节组织、中文字体回退、页边距和三线表思路；不复制其示例数值、固定三问结构或目录。

## 页面与结构

- [ ] A4 页面，四边约 2.5 cm；
- [ ] 摘要为第一页，页脚居中显示页码 1；
- [ ] 不生成目录，正文从摘要和关键词开始；
- [ ] 包含问题重述与分析、模型假设、符号与数据处理；
- [ ] 每问均有问题回应、模型建立、求解、结果解释、验证和适用边界；
- [ ] 包含模型评价、结论、真实参考文献和支撑材料清单；
- [ ] `paper/main.typ` 自动包含全部 `questions/*/paper.typ`。

## 国赛质量底线

- [ ] 变量、参数、单位、目标函数和约束首次出现即定义；
- [ ] 核心模型能脱离代码独立审阅；
- [ ] 统一评价口径下有简单基线或合理对照；
- [ ] 结果由 `result_ledger.json` / `results.typ` 绑定，不手抄关键数字；
- [ ] 图表服务于机制、比较、方案或边界解释，而不是堆验证痕迹；
- [ ] 说明算法停止条件、计算预算、误差来源、搜索范围和局限；
- [ ] 创新性用题意依据、相对基线的实际贡献和量化证据说明；
- [ ] 可读性检查具体页码、图表标题、段落目的和术语跳跃；
- [ ] AI 痕迹只报告低/中/高风险及证据，不写所谓“AI 率百分比”。

## 交稿前命令

```powershell
python scripts/contest.py verify <run_dir> --mode contest_standard
python scripts/contest.py package <run_dir>
```

工程命令通过只表示工程闭环通过，仍需按 `NATIONAL_CONTEST_REVIEW_WORKFLOW.md` 完成内容准入和独立评审。
