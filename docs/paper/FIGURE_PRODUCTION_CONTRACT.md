# 论文图表生产合同

## 目标

每张图必须服务一个明确结论，并绑定真实源数据、Claim、统计口径和导出文件。项目不根据正文中零散出现的数字重新拼图，也不生成只用于装饰的图。

## 后端分工

- Python 是最终投稿图的唯一绘制与导出后端，生成可编辑 SVG、PDF、600 dpi TIFF 和 PNG 预览。
- MATLAB 读取同一份 figure spec 与 CSV，独立复算每个序列的样本量、均值、标准差、最小值和最大值，并生成验证图。
- MATLAB 验证图不替换 Python 投稿图。两端统计量超过合同容差时，整个图表构建失败。

这种分工避免两个后端反复改变同一张最终图，同时保留独立数值复算。

## Figure spec

绘图前必须声明：

1. `core_conclusion`：图需要支持的一句话结论；
2. `archetype`：图的证据结构；
3. `purpose` 和 `claim_ids`：图回答的问题及支持的 Claim；
4. 源 CSV 的相对路径和 SHA-256；
5. x 轴、序列、单位、颜色、标记、线型和最终尺寸；
6. 样本量、中心统计量、离散程度与基线定义；
7. SVG/PDF/TIFF/PNG 导出合同；
8. MATLAB 复算容差与 Reviewer 风险。

## 视觉规则

- 白底、无图内大标题，图题由论文排版系统负责；
- 同一方法在不同图中保持颜色一致；
- 颜色不是唯一编码，同时使用线型或标记区分；
- SVG 文字保持为可编辑文本，PDF 使用 TrueType 字体；
- 图例、坐标、单位和统计口径必须在最终尺寸下可读；
- 每个 panel 必须承载独立证据，删除遮住后不影响结论的 panel。

## 输出

`figure_build_report.json` 记录源数据哈希、Python 最终输出哈希、两端统计量、最大数值差和最终状态。该报告证明图的可复现性与双端数值一致性，不证明论文结论已经通过独立审稿。
