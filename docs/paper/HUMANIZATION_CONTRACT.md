# 受控 Humanizer 合同

## 目标

受控润色只改善论文表达，不改变数学事实。输入和输出都必须经过保护字段检查；任一保护字段变化，润色结果立即作废并进入人工复核。

## 允许修改

- 调整句式和语序；
- 改善段落衔接；
- 合并重复表达；
- 删除没有信息量的模板化套话；
- 将过度列表化内容改为连贯学术段落。

## 禁止修改

- 数字和百分比；
- 单位、量纲和数量级；
- 公式、变量和上下标；
- 表格单元格值、图题、表题和图表编号；
- 文献编号和引用位置；
- 模型、算法和指标名称；
- 目标与约束方向；
- 结论正负方向、排序、最优性状态、最优性作用域和约束验证结论。

## 执行合同

1. 润色前提取数字、百分比、单位、公式片段、变量符号、引用编号、图表编号、表格值、方向短语、最优性作用域短语和锁定术语。
2. 只对普通语言片段改写，保护片段保持逐字一致。
3. 润色后重新提取并逐项比较。
4. 输出机器可读报告，不以“语义基本一致”替代逐项核对。
5. 任一 protected 字段发生变化时，`status` 必须为 `failed`。
6. 失败报告只能要求人工修订，不得自动把润色稿覆盖为原稿，也不得修改 Formal Result。

## 报告

每次受控润色都必须生成 `paper_humanization_report.json`，记录输入/输出 SHA-256、各类 protected 字段变化、改写段落数、移除套话数和最终状态。`status=passed` 只表示配置的差分保护通过，不表示论文事实已被独立验证。

## 人工复核

即使保护字段检查通过，涉及否定词、比较级、因果关系、最优性或敏感性结论的段落仍需人工复核。润色工具不得自动覆盖原论文。
