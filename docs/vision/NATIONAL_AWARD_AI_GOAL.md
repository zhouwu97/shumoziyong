# 国奖竞争力目标与能力边界

## 目标

构建一个能够在未知数学建模赛题上，以人工关键决策为边界，稳定产出完整、可复现、结论可追溯方案的 AI 协作系统，并通过盲测评估其是否具备全国奖竞争力。

这里的“国奖竞争力”是待验证的能力目标，不是当前项目名称或自动结论。系统不得因为模型数量、论文长度、代码复杂度或一次偶然高分而自称达到国奖水平。

## 可验证能力

每次正式运行必须可以追溯下列链路：

```text
未知题面与附件
-> 诊断、依赖图与路线选择
-> 已批准的执行合同
-> 候选执行记录
-> 干净环境独立重跑
-> 数值/数学专项验证
-> 正式结果与 Claim-Result Map
-> 论文、独立审稿与人工 Gate
```

“能完成”至少表示：题目拆分、基线、假设、数据缺口、路线淘汰条件、输入输出、随机性、失败降级和人工决策点均被显式记录。任何未被独立重跑或验证的执行器输出都只能是 `candidate result`，不能写入正式结论。

## 不可妥协的约束

- AI 不得伪造命令执行、数据、参数、图表或比较结果。
- 论文只能组织已验证的事实；缺少证据的结论必须标记为缺失或不确定。
- 启发式解不得表述为全局最优，相关性不得表述为因果。
- Executor 不得修改已批准的执行合同，也不得自行宣布 Gate 通过。
- 同一执行 Agent 不能单独确认自己的结果；正式结果由 Collector 和 Validator 决定。
- 能力成熟度只能由 `capability_evidence` 和政策现场派生，禁止人工手填状态。

## 资格标准

单个 Profile 进入 `profile_qualified` 前，至少需要三道独立题、跨两个年份和两个机制类别，完整重跑率 100%，无结果伪造、无致命数学错误，并经两位独立评审确认没有共同 P0。

系统进入 `competition_ready` 前，至少完成两次受限时完整模拟赛；进入 `national_award_competitive` 前，除上述证据外，还必须具有正式国家级奖项，或满足政策规定的多题型盲测和独立专家盲评门槛。详细机器判定规则由 [能力成熟度政策](../../policies/capability_maturity_policy.json) 定义。
