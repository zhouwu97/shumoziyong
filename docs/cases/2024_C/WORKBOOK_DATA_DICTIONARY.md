# 2024-C 工作簿字段字典

## 规范化规则

1. 工作表名称按官方中文名称精确匹配。
2. 附件 1 的 `说明` 和附件 2 的合并单元格字段在读取时前向填充；原始文件不改写。
3. 地块 ID、作物编号、季次和地块类型组成稳定键；空白不是未知值的默认许可，无法恢复时必须报错。
4. 作物名称中的尾随空格（如 `菠菜 `、`生菜 `）保留原始值，同时在规范化字典中建立去空格显示名。
5. 数值字段使用 IEEE-754 float64 读取；单位转换必须显式记录。

## 附件 1：地块与作物基础数据

### `乡村的现有耕地`

| 列 | 字段 | 类型 | 单位/语义 |
| --- | --- | --- | --- |
| A | `地块名称` | string | 唯一地块 ID，如 `A1`、`F4` |
| B | `地块类型` | enum | 平旱地、梯田、山坡地、水浇地、普通大棚、智慧大棚 |
| C | `地块面积/亩` | number | 地块可用面积，亩 |
| D | `说明` | text | 合并单元格规则和季次说明；D2:D55 合并，需前向填充 |

观测结构：54 行地块记录；类型计数为平旱地 6、梯田 14、山坡地 6、水浇地 8、普通大棚 16、智慧大棚 4。露天地块为 34 条，大棚为 20 条。

### `乡村种植的农作物`

| 列 | 字段 | 类型 | 单位/语义 |
| --- | --- | --- | --- |
| A | `作物编号` | integer | 1–41 的作物 ID；末行是说明文本，不是作物 |
| B | `作物名称` | string | 官方作物名 |
| C | `作物类型` | enum/text | 粮食、粮食（豆类）、蔬菜、蔬菜（豆类）、食用菌 |
| D | `种植耕地` | text | 适宜地块和季次；D2:D16、D18:D35、D36:D38、D39:D42 合并 |
| E | `说明` | text | 季次和管理规则；E2:E42 合并 |

规范化时必须保留作物 ID 与原始名称映射，不能仅按显示名称连接。

## 附件 2：历史种植与统计

### `2023年的农作物种植情况`

| 列 | 字段 | 类型 | 单位/语义 |
| --- | --- | --- | --- |
| A | `种植地块` | string | 地块 ID；A 列存在跨行合并，需前向填充 |
| B | `作物编号` | integer | 连接附件 1 作物字典 |
| C | `作物名称` | string | 用于交叉核对，不作为唯一键 |
| D | `作物类型` | string | 历史记录类型 |
| E | `种植面积/亩` | number | 2023 年实际种植面积 |
| F | `种植季次` | enum | 单季、第一季、第二季 |

原始工作表 88 行（含表头和合并展开空行），前向填充后得到 87 条作物种植明细；其中覆盖 54 个地块分组，总面积 1212.1 亩。Validator 必须保留作物明细，不得把同一地块的多作物行压成一条记录。

### `2023年统计的相关数据`

| 列 | 字段 | 类型 | 单位/语义 |
| --- | --- | --- | --- |
| A | `序号` | integer | 统计行序号 |
| B | `作物编号` | integer | 作物 ID |
| C | `作物名称` | string | 交叉核对字段 |
| D | `地块类型` | enum | 统计适用地块类型 |
| E | `种植季次` | enum | 单季、第一季、第二季 |
| F | `亩产量/斤` | number | 每亩产量 |
| G | `种植成本/(元/亩)` | number | 每亩成本 |
| H | `销售单价/(元/斤)` | string/number | 区间如 `2.50-4.00` 或固定值；区间使用端点中点必须显式记录 |
| I–J | 预留列 | empty | 不得推断为额外参数 |

原始工作表 111 行，108 条统计记录；相同作物—季次在官方数据中应保持价格一致，Validator 需检查这一不变量。
