# 2024-C Q2 不确定性模型合同

## 1. 合同状态与边界

本合同冻结 Q2 的不确定性语义，供后续 Solver、Formal Result 和独立 Validator 共同使用。它不声明 Solver 已完成、不产生 `result2.xlsx`，也不覆盖 Q3 的相关性、替代性或互补性模型。

```yaml
problem_id: 2024-C
subproblem_id: Q2
contract_id: 2024c-q2-uncertainty-v1
status: model_contract_frozen_solver_pending
proxy_data_used: false
qualification_claimed: false
```

## 2. 决策与硬约束

决策是一个 2024--2030 年的**非自适应**种植方案 `x[p,t,s,c]`。方案在不观察未来随机参数的情况下确定；所有情景使用同一组面积。Q2 沿用 Q1 的地块容量、作物适宜性、季次制度、实际相邻季重茬、三年豆类窗口和管理性最小活动面积约束。Q2 不把随机参数写入硬约束，也不允许通过改变情景中的方案掩盖约束违规。

产量超过该情景销售量的部分按 Q1 的“滞销、收入为 0”口径处理。该口径是保守的 Q2 风险基线；折价口径仍属于 Q1 的独立情景，不与 Q2 混合。

## 3. 随机参数与来源

所有不确定量均以 2023 年附件 2 的对应值为基准。除特别注明外，年度因子在作物、年份和参数之间相互独立；同一参数的年度因子也相互独立。Q2 不使用相关矩阵，相关性只在 Q3 合同中定义。

| 参数 | 作物类别 | 年度生成规则 | 备注 |
| --- | --- | --- | --- |
| 预期销售量 | 小麦、玉米 | `S[t] = S[t-1] * (1 + U(0.05, 0.10))` | 年增长率每年重新采样 |
| 预期销售量 | 其他作物 | `S[t] = S[t-1] * (1 + U(-0.05, 0.05))` | 年变化每年重新采样 |
| 亩产量 | 全部作物 | `Y[t] = Y[2023] * U(0.90, 1.10)` | 各年相对 2023 基准独立采样 |
| 种植成本 | 全部作物 | `C[t] = C[t-1] * (1 + U(0.045, 0.055))` | “平均年增长 5%”的可复算区间化解释 |
| 销售价格 | 粮食类 | `P[t] = P[2023]` | 题面“基本稳定”按不变处理 |
| 销售价格 | 蔬菜类 | `P[t] = P[t-1] * (1 + U(0.045, 0.055))` | 年增长率每年重新采样 |
| 销售价格 | 食用菌（羊肚菌除外） | `P[t] = P[t-1] * (1 - U(0.01, 0.05))` | 年降幅每年重新采样 |
| 销售价格 | 羊肚菌 | `P[t] = P[t-1] * 0.95` | 题面明确每年下降 5% |

`U(a,b)` 为包含端点的连续均匀分布。类别映射必须来自附件 1 的官方作物字典，不得按作物编号范围猜测；无法映射时 Validator 必须失败。

## 4. 情景、随机种子与重复实验

使用 NumPy `PCG64` 伪随机生成器。主实验固定 `seed=20240724`、`scenario_count=256`；情景按 `scenario_0000` 至 `scenario_0255` 编号。每个情景完整生成 2024--2030 的全部不确定参数后再计算利润。

为报告重复性，另使用 `replicate_seeds=[20240724,20240725,20240726,20240727,20240728]`，每个种子生成 256 个情景。Solver 只用主种子产生方案；其余四组和主种子的未参与求解情景仅用于复核稳定性，不能重新选择方案。

Formal Result 必须保存：合同版本、生成器、种子列表、情景数量、每个情景参数摘要或其 SHA-256、方案 SHA-256 以及运行日志。Validator 必须按相同规则重新生成情景并逐项复算，禁止接受手工填写的均值或区间。

## 5. 风险目标与报告统计

情景利润为：

`profit_s(x) = 销售收入_s(x) - 种植成本_s(x)`。

以损失 `loss_s(x) = -profit_s(x)` 定义 `CVaR_0.90(loss)`，采用有限样本的标准上尾平均：先取损失从大到小排序，取最差 `ceil(0.10 * 256)=26` 个情景的平均值。主种植方案最大化：

`mean(profit_s) - 0.25 * CVaR_0.90(loss_s)`。

`0.25` 是本合同的风险厌恶系数，不是官方题面参数，必须在结果中显式标注，并至少以 `0`、`0.25`、`0.50` 做敏感性重算。所有报告至少包含均值、标准差、P05、P50、P95、最坏情景利润、CVaR，以及相同情景下 Q1 基线的对应统计量和配对差异。

## 6. Validator 复算边界

Q2 Validator 必须独立读取官方附件和本合同，校验：

1. 合同身份、参数类别映射、随机生成器、种子、情景数量和情景编号；
2. 每个情景的销量、亩产、成本、价格因子及其 SHA-256；
3. 方案面积、Q1 全部硬约束和非负有限性；
4. 每个情景的产量、销售收入、成本、利润和超产处理；
5. 均值、分位数、最坏情景、CVaR 和风险目标；
6. Q1 基线使用同一情景集的比较结果；
7. `result2.xlsx` 的文件身份、模板版式、单元格与 Formal Result 方案的一致性。

Validator 不运行候选 Solver，不把 `status`、`objective_reported` 或人工统计值当作真值；任一情景、哈希、约束或统计量不一致即失败。Q2 在独立 Workbook Validator 和 Formal Result 全部通过前，不得登记为完成或生产就绪。
