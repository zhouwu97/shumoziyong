# 2023-B Validator 公式复核 Pilot

日期：2026-07-13

Pilot：`2023b_validator_formula_pilot_v2`

结论：旧 `a092_2023b_q1_q2_v1` 的问题二水深公式缺少方向投影，不能继续用于确认性实验；经题面、图示、闭式公式、三维射线求交和故障注入交叉验证后，`2023b_q1_q2_v2` 公式合同通过 Pilot。

## 1. 证据边界

本次只使用官方题面 `official_materials/2023_B/problem/B题.pdf` 第 2–3 页、空白结果模板和仓库内第一轮实验输出。没有使用同题优秀论文、参考答案或题解作为真值。

v1 Validator 保留不改，用于解释历史实验；v2 使用独立目录 `validators/problem_boundary_v2/`。本报告冻结的是 2023-B 公式合同，不等同于完整 `a092_confirmatory_v2` 协议冻结。

## 2. 题面坐标口径

图 7 支持问题一的固定约定：距离正向指向浅水侧，水深向下为正，因此

```text
D(x) = 70 - x tan(alpha)
```

图 8 明确 `beta` 是测线方向与坡面向上法向在水平面投影的夹角。向上法向的水平投影指向坡面降低的一侧，即深水方向，因此固定：

```text
beta = 0 deg：沿深水方向离开中心
beta = 180 deg：沿浅水方向离开中心
```

若选用相反法向，表 2 的 0°/180°、45°/225°、90°/270°、135°/315° 会整体互换。v2 不允许隐式选择相反法向，Runtime 和 Validator 必须共享上述显式约定。

## 3. 冻结公式

令 `gamma=theta/2`，`s` 的单位为海里。

沿测线的坡度投影控制船位水深：

```text
D(s,beta) = 120 + 1852 s tan(alpha) cos(beta)
```

测线法向截面内的横坡控制波束与坡面的交点：

```text
delta = atan(tan(alpha) sin(beta))
```

覆盖宽度按海底交线长度计算：

```text
W = D sin(gamma) [sec(gamma+delta) + sec(gamma-delta)]
```

问题一的相邻条带宽度不同，不能继续使用平底近似 `1-d/W_current`。v2 先计算前一条带浅水侧半宽、当前条带深水侧半宽和坡面上的测线间距，得到有符号交叠长度，再除以前一条条带宽度。负值表示漏测。

## 4. 手算与独立复算点

闭式公式没有调用 Validator 的内部结果作为第二真值。Pilot 另用三维射线参数方程与坡面方程直接求交，再计算两个边缘交点之间的三维距离。

| beta | 距离/海里 | 水深/m | 覆盖宽度/m | 射线求交最大差 |
|---:|---:|---:|---:|---:|
| 0° | 0.3 | 134.548898024 | 466.091054959 | 0 |
| 45° | 0.6 | 140.575248903 | 487.551891549 | 1.14e-13 |
| 90° | 2.1 | 120.000000000 | 416.691869935 | 5.68e-14 |
| 180° | 2.1 | 18.157713833 | 62.900165817 | 0 |

结构检查也通过：`beta=90°/270°` 时沿线水深恒为 120 m；`beta=0°` 随距离变深；`beta=180°` 随距离变浅。

## 5. 705.584056 的根因

v1 对表 2 的所有方向都使用：

```text
D_old(s) = 120 - 1852 s tan(alpha)
```

因此在 2.1 海里处，它对每一行都得到约 18.1577 m 水深和约 62.9002 m 覆盖宽度。正确方向模型在相反坡向得到约 221.8423 m 水深和约 768.4842 m 覆盖宽度：

```text
768.484221817 - 62.900165817 = 705.584056000
```

四次运行几乎相同的最大偏差由这一缺失的 `cos(beta)` 水深投影直接造成，不是候选解随机误差。
