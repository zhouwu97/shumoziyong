# 2024-C 农作物种植模型规格

## 决策变量与目标

令 `x[p,t,s,c]` 为地块 `p` 在年份 `t`、季次 `s` 种植作物 `c` 的面积（亩），`z[p,t,s,c]` 为对应活动变量。水浇地另设二元模式变量，在“单季水稻”和“两季蔬菜”之间互斥。正常销售量按 `year + crop_id + season` 聚合，跨地块类型合并。

Q1 最大化 2024--2030 年总利润。滞销情形的超产收入为 0；折价情形的超产收入为中点价格的 50%。求解器内部仅使用轻量碎片化惩罚打破同利润并列，正式利润不包含该惩罚，并由独立 Validator 从官方结果表复算。

HiGHS 在时限内若返回 incumbent，Solver 会重新检查变量边界、整数性和全部线性约束后再导出，并显式记录 `solver_status`、`mip_gap` 与 `optimality_proven`。时限内可行解只能登记为“可行基线”，不得表述为已证明最优；官方工作簿还必须通过不导入 Solver 的独立 Validator。

## 约束口径

1. 每地块、年份、有效季次的面积不超过附件 1 容量。
2. 地块类型、季次和作物必须满足附件 1 的七条适种说明。
3. 水浇地每年只能选择单季水稻或两季蔬菜模式。
4. 同一地块的实际相邻季次不得重复同一作物；含 2023 到 2024 的边界。
5. 每个地块在 2023--2025 至 2028--2030 的全部滚动三年窗口中，豆类累计面积不少于地块面积。
6. 面积和正常销售量非负，模式与活动变量为二元变量。

## 管理性假设

题面只定性要求种植面积“不宜太小”、种植地“不宜太分散”，没有给出阈值。当前基线固定：单次活动面积不少于 `max(0.3 亩, 地块面积的 10%)`，并对每个活动地块--作物组合施加 1 元并列排序惩罚。两项均在 Solver Settings、Formal Result 和 Validator 中显式记录；后续灵敏度分析必须改变阈值后重求解，不能把该参数表述为官方事实。

## 重茬语义

“连续重茬”按真实相邻种植季解释。旱地的相邻年份单季、水浇地的相邻单季水稻、智慧大棚的第一季到第二季及第二季到下一年第一季需要显式检查。水浇地两季蔬菜、普通大棚蔬菜与食用菌的相邻季作物集合不重叠，因此不会出现同作物连续重茬；不额外禁止经过另一类作物间隔后的同作物复种。

## 单位

- 产量与销量：斤
- 亩产量：斤/亩
- 成本：元/亩
- 价格：元/斤
- 目标：元
