# 提示词版本日志

## 记录规则

每次修改提示词必须新增一条记录。不要只写“优化了提示词”，必须说明来源、修改点、测试结果和是否稳定。

## 版本状态

| 状态 | 含义 |
|---|---|
| draft | 草稿，未测试 |
| testing | 正在旧题测试 |
| stable | 至少通过 2 道同类旧题测试 |
| rollback | 已回滚 |
| frozen | 赛前冻结，不再大改 |

## 稳定版判定标准

1. 同类旧题测试总控诊断 >= 80 分；
2. 关键失败标签不超过 1 个；
3. 不出现 P3 模型不匹配、P4 高级模型滥用、P5 数据需求不清；
4. 输出中必须包含人工确认项；
5. 至少通过 2 道同类旧题测试后，才能标记为 stable。

## 回滚规则

如果新版测试分数低于旧版，或新增 P3/P4/P5 关键失败标签，则回滚到上一稳定版本，并在本日志中标记 rollback。
